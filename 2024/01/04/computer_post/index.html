数据结构

绪论

  • 数据结构三要素
    • 逻辑结构：集合、线性结构、树形结构、图状结构
    • 存储结构：顺序存储、链式存储、索引存储、散列存储
    • 数据的运算
  • 算法的重要特性：有穷性、确定性、可行性、输入、输出
  • “好”的算法：正确性、可读性、健壮性、高效率与低存储量需求
  • 时间复杂度：基本运算的数量级
    • 最好、平均、最坏
    • 加法规则、乘法规则
  • 空间复杂度需要考虑：各种变量所占空间（如辅助数组）、递归工作栈深度
  • 算法原地工作：\(S(n)=O(1)\)

线性表

  • 线性表：相同类型数据元素、\(n\ge 0\)、有限序列

    • 除第一个元素外，每个元素有且仅有一个直接前驱
    • 除最后一个元素外，每个元素有且仅有一个直接后继

  • 注意区分位序（从1开始）与数组下标（从0开始）

  • 需要将修改结果“带回来”则要使用指针

  • 顺序表的插入删除后记得修改长度变量

  • 链表：单链表、双链表、循环单链表、循环双链表、静态链表

  • 引入头结点的优点

    • 第一个位置与其他位置上的操作统一

    • 空表和非空表的操作统一

  • 链表的逆置

  • 时间复杂度为\(O(n)\)，空间复杂度为\(O(1)\)

  • 静态链表：\(0\)号结点作为头结点、next=-2表示空结点（初始化）、next=-1表示NULL（最后一个结点的游标）、不能随机存取、最大容量固定、应用：文件分配表FAT

  • 链栈（链表表头为栈顶）

  • 栈的应用：括号匹配、表达式求值、递归

  • 队列：先进先出\(LIFO\)

  • 循环队列区分队空还是队满的三种方式

    • 牺牲一个单元

    • 类型中增设表示元素个数的数据成员size

      • size=0表示队空

      • size=MaxSize表示队满

    • 类型中增设tag数据成员

      • 初始时tag=0

      • 删除元素之后置tag=0，若front==rear && tag==0则表示队空

      • 插入元素之后置tag=1，若front==rear && tag==1则表示队满

    • 队满条件：front == (rear + 1) % MaxSize
    • 队空条件：front == rear
    • 队列中元素的个数：(rear - front + MaxSize) % MaxSize

  • 链队列：头指针指向队头结点、尾指针指向队尾结点

  • 链队列的操作要着重考虑仅含0个、1个元素的特殊情况（可能队头队尾指针都要修改）

  • 队列的应用：逐层问题、主机与外设速度不匹配（数据缓冲区）、多用户的资源竞争（先来先服务\(FCFS\)）

查找

  • 折半查找的判定树是一种特殊的平衡二叉排序树

    • 不仅平衡因子的绝对值\(\le 1\)，而且每棵子树的\(\text{左子树结点数}-\text{右子树结点数}=0/-1\)（当取\(mid=\left\lfloor \frac{low+high}{2}\right\rfloor\)时）
    • 例如，不能作为判定树
    • 即分支结点的\(\text{左子树结点数}-\text{右子树结点数}\)的值要么取\(0/1\)，要么取\(0/-1\)

  • 分块查找（索引顺序查找）

    • 块内无序，块间有序
    • 折半查找索引表时，当\(low>high\)时，还需在\(low\)所指分块中顺序查找
    • 长度为\(n\)的查找表均匀地分为\(\sqrt{n}\)块，每块\(\sqrt{n}\)个记录，平均查找长度取最小值\(\sqrt{n}+1\)

  • 对二叉排序树进行中序遍历，可以得到一个递增的有序序列

  • 平衡二叉树

    • 结点的平衡因子=左子树高度-右子树高度，取值范围为{-1,0,1}
    • 深度为\(h\)的AVL树中的最少结点数为\(n_h\)，最多结点数为\(2^h-1\)
    • \(n_0=0,n_1=1,n_2=2,n_h=n_{h-1}+n_{h-2}+1\)

  • 红黑树

    • 左根右、根叶黑、不红红、黑路同
    • 从某结点出发（不含该结点）到达一个叶结点的任意一个简单路径上的黑结点总数称为该结点的黑高（记为bh）
    • 从根结点到叶结点的最长路径不大于最短路径的二倍
    • 有\(n\)个内部结点的红黑树的高度\(h\le 2\log_2(n+1)\)
    • 若根结点黑高为\(h\)，则内部结点数最少为\(2^h-1\)（全黑满树），最多为\(2^{2h}-1\)（黑红层次交替）

      • 最多经过m-1次探测就会遍历表中所有位置

排序

  • 对于任意n个关键字的序列进行基于比较的排序，至少需要的比较次数为\(\left\lceil\log_2(n!)\right\rceil\)

  • 一趟排序：对所有未确定最终位置的元素进行一遍处理

    • 一次划分：只能确定一个元素的最终位置
    • 一趟排序：可以确定多个元素的最终位置（例如，“标准”快速排序的第二趟排序可以确定两个元素的最终位置）

  • 大根堆

    • 初始化
      • 按分支结点编号从大到小（\(\left\lfloor\frac{n}{2}\right\rfloor\to 1\)），逐个调整
        • 若当前结点不满足根\(\ge\)左、右，则将当前结点与较大的孩子互换
        • 若互换破坏了下一级的堆，则用相同的方法向下调整
    • 插入：先将该结点放在堆的末端，再对该结点向上执行调整操作（沿着新结点到根结点的路径逐步向上调整）
    • 删除：输出堆顶元素后，先用最后一个元素填补根位置，再对根结点向下执行调整操作

  • 堆排序

    • 空间复杂度：\(O(1)\)
    • 时间复杂度：\(O(n\log_2n)\)（建堆时间为\(O(n)\)，之后有（n-1）次向下调整操作，每次调整的时间复杂度为\(O(h)=O(\log_2n)\)）

  • 对各数位有优先级的关键字进行基数排序，只能按照最低位优先\(LSD\)的方法

  • 基数排序

    • 时间复杂度为\(O(d(n+r))\)（\(n\)个\(d\)元组关键字、\(r\)为基数）（\(d\)趟分配和收集：一趟分配需要\(O(n)\)、一趟收集需要\(O(r)\)）
    • 空间复杂度为\(O(r)\)（\(r\)个队列：\(r\)个队头指针、\(r\)个队尾指针）

  • k路归并的败者树深度为\(\left\lceil\log_2k\right\rceil\)

  • 最佳归并树

    • 归并过程中的磁盘\(I/O\)次数=\(2\times WPL\)
    • 保证\((\text{初始归并段数}n_0+\text{虚段数}-1)\%(\text{归并路数}k-1)==0\)

AVL树的删除操作

  1. 删除该结点（方法同二叉排序树的删除操作）
    • 若被删除结点是叶结点，则直接删除
    • 若被删除结点只有一个子树，则用该子树填补删除位置
    • 若被删除结点有两棵子树，则用被删除结点的直接前驱（或后继）结点填补被删除位置，并转换为对该直接前驱（或后继）结点的删除操作
  2. 向上查找最小不平衡子树，若查找失败则结束
  3. 查找最小不平衡子树下高度最大的子树和子子树的根结点（记为儿子结点和孙子结点）（高度相同时任取其一）
  4. 根据孙子结点相对最小不平衡子树根结点的位置，调整平衡
    • 孙子结点在LL位置：儿子结点右单旋
    • 孙子结点在RR位置：儿子结点左单旋
    • 孙子结点在LR位置：孙子结点先左单旋，再右单旋
    • 孙子结点在RL位置：孙子结点先右单旋，再左单旋
  5. 如果不平衡向上传导，则继续2

置换选择排序

设初始待排文件为FI，初始归并段输出文件为FO，内存工作区为WA，FO和WA的初始状态为空，WA可容纳w个记录。置换-选择算法的步骤如下：

  1. 从FI输入w个记录到工作区WA
  2. 从WA中选出其中关键字取最小值的记录，记为\(MINIMAX\)记录（需利用败者树来实现）
  3. 将\(MINIMAX\)记录输出到FO中
  4. 若FI不空，则从FI输入下一个记录到WA中
  5. 从WA中所有关键字比\(MINIMAX\)记录的关键字大的记录中选出最小关键字记录，作为新的\(MINIMAX\)记录
  6. 重复(3)$\((5)，直至在WA中选不出新的\)MINIMAX$记录为止，由此得到一个初始归并段，输出一个归并段的结束标志到FO中
  7. 重复(2)$$(6)，直至WA为空。由此得到全部初始归并段

      • 计数器的变化规则
        • 命中时，比其低的计数器加一，所命中的行的计数器清零，其余不变
        • 未命中且还有空闲行时，新装入的行的计数器置0，其余全部加一

    • 写不命中
      • 写分配法：加载主存中的块到Cache中，然后更新此Cache块（暂不更新主存块，因此常与回写法合用）
      • 非写分配法：只写入主存，不进行调块

  • 实地址（物理地址）、实地址空间（主存地址空间）

  • CPU使用虚地址时，由辅助硬件找出虚地址与实地址之间的对应关系，并判断该虚地址对应的存储单元时候已经装入内存

    • 若已在内存中，则通过地址变换，CPU可直接访问主存指示的实际单元
    • 若不在主存中，则把包含该字的页或段调入主存后再由CPU访问
    • 若主存已满，则采用替换算法置换主存中的交换块（页面）

  • 虚拟存储机制采用全相联映射和回写法

  • 虚拟空间和主存空间被划分为同样大小的页

    • 主存的页称为实页、页框
    • 虚存的页称为虚页

  • 虚拟地址分为两个字段：虚页号 页内地址

  • 页表：一张存放在主存中的虚页号和实页号的对照表，一般长久地保存在内存中（慢表）

  • 页表表项

    • 有效位：为1表示该虚拟页已从外存调入主存
    • 脏位（修改位）：=1表示该页被修改过，被替换时需要写回磁盘
    • 引用位（使用位）：用来配合替换策略进行设置
    • 物理页号

  • 页表基址寄存器存放进程的页表首地址

  • 虚拟地址变换为物理地址

    1. 根据虚拟地址的高位部分虚页号和页表基址寄存器的内容找到对应的页表项
    2. 若装入位为1，则取出物理页号，与虚拟地址的低位部分页内地址拼接形成物理地址物理页号 页内地址
    3. 若装入位为0，则说明缺页，需要OS进行缺页处理

  • 页式虚拟存储器

    • 优点：页面的长度固定，页表简单，调入方便

    • 缺点：平均每个进程有半页的内部碎片，处理、保护和共享不如段式虚拟存储器方便

  • 快表TLB
    • 将一段时间内总是经常访问的页对应的页表项存放在高速缓冲器组成的快表TLB中
    • 通常采用全相联或组相联方式
    • 每个TLB项由页表表项内容加上一个TLB标记字段组成
    • TLB标记字段用于表示该表项取自页表中哪个虚页号对应的页表项，其内容为
      • 全相联方式下为该页表项对应的虚页号
      • 组相联方式下为对应虚页号的高位部分（低位部分用于组索引）

    • 单指令周期：对所有指令都选用相同的执行时间来完成、每条指令都在一个时钟周期内完成
    • 多指令周期：可选用不同个数的时钟周期来完成不同指令的执行过程、下一条指令只能在前一条指令执行完成后才能启动
    • 流水线方案：指令之间可以并行执行、在每个时钟周期启动一条指令、尽量让多条指令同时运行，但各自处于不同的执行步骤中

    • CPU内部多总线方式：效率比上者高
    • 专用数据通路方式：性能较高、但硬件量大

  • 微程序控制器的工作过程

    1. 执行取微指令公共操作
      • 在机器开始运行时，自动将取指微程序的入口地址送入CMAR，并从CM中取出相应的微指令送入CMDR
      • 取指微程序的入口地址一般为CM的0号单元
      • 当取指微程序执行完后，从主存中取出的机器指令就已存入指令寄存器中
    2. 由机器指令的操作码字段通过微地址形成部件产生该机器指令所对应的微程序的入口地址，送入CMAR
    3. 从CM中逐条取出对应的微指令并执行
    4. 执行完对应于一条机器指令的一个微程序后，回到取指微程序的入口地址，继续第1步，以完成取下一条机器指令的公共操作

    • 时间上的并行技术（不同阶段在不同的功能部件上并行指令）：流水线技术
    • 空间上的并行技术（设置多个相同任务的功能部件）：超标量处理机

  • 五级流水线举例

    • 取指IF：从指令存储器或Cache中取指令

    • 译码ID/读寄存器：操作控制器对指令进行译码，同时从寄存器堆中取操作数

    • 执行EX/计算地址：执行运算操作或计算地址

    • 访存MEM：对存储器进行读写操作

    • 写回WB：将指令执行结果写回寄存器堆

  • 流水线设计的原则

    • 指令流水段的个数以最复杂指令所用的功能段个数为准

    • 流水段的长度以最复杂的操作所花的时间为准

  • 为了利于指令流水线，指令集应具有如下特征

    • 指令长度应尽量一致
    • 指令格式应尽量规整
    • 采用Load/Store指令
    • 数据和指令在存储器中“对齐“存放

  • 流水线方式并不能缩短单条指令的执行时间，但对于整个程序来说，执行效率得到了大幅提高

  • 流水线的冒险与处理
冒险 含义 解决
结构冒险（资源冒险） 多条指令在同一时刻争用同一资源，即由硬件资源竞争造成的冲突 后一条相关指令（及其后续指令）暂停一个时钟周期、单独设置数据存储器和指令存储器
数据冒险 下一条指令会用到当前指令计算出的结果：写后读RAW相关、读后写WAR相关、写后写相关WAW 把遇到数据相关的指令及其后续指令都暂停一至几个时钟周期，直到数据相关问题消失后再继续执行、设置相关专用通路（数据旁路技术）、通过编译器对相关指令编译优化，调整指令顺序
控制冒险 遇到转移、调用或返回等指令会改变PC值，造成断流 分支预测，尽早生成转移目标地址、预取转移成功和不成功的目标指令、加快和提前形成条件码、提高转移方向的猜准率
  • 数据旁路技术：不等前一条指令把计算结果写回寄存器组，下一条指令指令直接把前一条指令的ALU的计算结果作为输入数据，开始计算，使本来需要暂停的操作变得可以继续执行
  • Cache缺失的处理过程也会引起流水线阻塞
  • 流水线的性能指标
    • 流水线吞吐率\(TP=\frac{n}{T_k}=\frac{n}{(k+n-1)\Delta t}\)（n为任务数、\(T_k\)为处理完n个任务所用的总时间、k为流水线段数、\(\Delta t\)为时钟周期）
    • 流水线加速比\(S=\frac{T_0}{T_k}=\frac{kn\Delta t}{(k+n-1)\Delta t}=\frac{kn}{k+n-1}\)（\(T_0\)为不使用流水线的总时间、\(T_k\)为使用流水线的总时间）
  • 增加指令级并行的策略
    • 多发射技术：采用多个内部功能部件，使流水线功能段能同时处理多条指令，处理机一次可以发射多条指令进入流水线执行
    • 超流水线技术：通过增加流水线段数来使更多的指令同时在流水线中重叠执行
  • 超标量流水线技术（动态多发射技术）
    • 每个时钟周期可并发多条独立指令
    • 需配置多个功能部件
    • CPI<1
  • 超长指令字技术（静态多发射技术）
    • 编译程序挖掘指令间潜在的并行性
    • 将多条能并行操作的指令组合成一条具有多个操作码字段的超长指令字
    • 需要采用多个处理部件
  • 超流水线技术
    • 划分更多的流水段（时钟周期也会随之变短，即始终保持一个时钟周期一个流水段）、提高流水线主频
    • CPI=1
  • 基于指令流的数量和数据流的数量划分
    • 单指令流单数据流SISD结构
    • 单指令流多数据流SIMD结构
      • 一个控制部件、多个处理单元（执行同一条指令，但有不同的数据地址）
      • 向量处理器：直接操作一维数组（向量）
      • 数据级并行
    • 多指令流单数据流MISD结构
      • 实际上不存在
    • 多指令流多数据流MIMD结构
      • 并行程度更高的线程级并行或线程级以上并行计算模式
      • 多计算机系统（可理解为分布式计算机系统）：各自独立的地址空间；多处理器系统（共享存储多处理器SMP）：共享的单一地址空间

总线

  • 特点：分时和共享
  • 总线设备：主设备（获得总线控制权）、从设备
  • 特性：机械、电气、功能、时间
  • 分类：片内总线（芯片内部）、系统总线（各功能部件之间）、通信总线
  • 系统总线的结构：单总线结构、双总线结构（主存总线、I/O总线）、三总线结构（主存总线、I/O总线、DMA总线）
  • 总线性能指标
    • 总线传输周期（一次总线操作所需的时间，通常包括若干总线时钟周期）、总线工作频率
    • 总线时钟周期（机器的时钟周期）、总线时钟频率
    • 总线宽度：数据总线根数
    • 总线带宽：\(\text{总线带宽}=\text{总线宽度}\times\text{总线(工作)频率}\)
    • 总线复用：一种信号线在不同的时间传输不同的信息
    • 信号线数：地址总线、数据总线、控制总线的总根数
  • 一般情况下，（主存）存储周期>总线传输周期>（主存）存取时间
  • 总线定时：总线在双方交换数据的过程中需要时间上配合关系的控制
  • 同步定时方式
    • 系统采用一个统一的时钟信号来协调发送和接收双方的传送定时关系
    • 时钟产生相等的时间间隔，每个间隔构成一个总线周期
    • 一个总线周期进行一次数据传送
  • 异步定时方式：依靠传送双方相互制约的“握手”信号来实现定时控制
    • 不互锁方式、半互锁方式、全互锁方式

  • 猝发（突发）传输
    • 在一个总线周期中，可以传输多个存储地址连续的数据，即一次传输一个地址和一批地址连续的数据
    • 常用于主存-Cache之间的“块”传输，即通常主存总线可以支持猝发传输方式
  • 有的总线采用信号线复用技术，即一种信号线在不同的时间传输不同的信息

      • 可以用统一的访存指令（如Load/Store指令）访问I/O端口

      • 需要设置专门的I/O指令来访问I/O端口

  • 程序中断方式
    • CPU与外设并行工作、传送与主程序串行工作

  • DMA方式

    • 主存与DMA接口之间有一条直接数据通路

    • DMA方式传送数据不需要经过CPU，因此不必中断现行程序

    • I/O与主机并行工作，程序和传送并行工作

    • 主存既可被CPU访问，又可被外设访问

    • 在数据块传送时，主存地址的确定、传送数据的计数等都由硬件电路直接实现

    • 主存中要开辟专用缓冲区，及时供给和接收外设的数据

    • DMA传送速度快，CPU和外设并行工作，提高了系统效率

    • DMA在传送开始前要通过程序进行预处理，结束后要通过中断方式进行后处理

  • DMA使用主存的方式

    • 停止CPU访存：CPU停止访问主存，直到DMA传送一块数据结束
    • 周期挪用（周期窃取）：
      • CPU不在访存，则DMA直接访存
      • CPU正在访存，则待存取周期结束后，CPU让出总线占有权
      • I/O和CPU同时请求访存，出现访存冲突，则CPU暂时放弃总线占有权
        • I/O访存优先级高于CPU访存，因为I/O不立即访存就可能丢失数据
    • DMA与CPU交替访存：将CPU周期分为两个周期，某一周期专供DMA访存，另一周期专供CPU访存

  • DMA的传送过程：预处理、数据传送、后处理

中断请求

  • 通过INTR线发出的是可屏蔽中断，通过NMI线发出的是不可屏蔽中断
  • 不可屏蔽中断用于处理紧急和重要的事件，如时钟中断、电源掉电等，其优先级最高
  • 内部异常，即使在关中断模式下也会被响应
  • 可屏蔽中断的优先级最低，在关中断模式下不会被响应

中断响应判优

  • 中断响应优先级：CPU响应中断请求的先后顺序
  • 通过硬件排队器实现
  • 不可屏蔽中断>内部异常（终止>故障、自陷）>可屏蔽中断
  • DMA中断请求>I/O设备传送的中断请求
  • 中断优先级包括中断响应优先级（在硬件线路上是固定的）和中断处理优先级（可利用中断屏蔽技术动态调整，以实现多重中断）

CPU响应中断的条件

  • 中断源有中断请求
  • CPU允许中断及开中断（异常和不可屏蔽中断不受此限制）
  • 一条指令执行完毕（异常不受此限制），且没有更紧迫的任务

中断处理过程

  1. 关中断
    • 置“中断允许”IF触发器为0
    • 保证在保存断点和程序状态时不被中断
  2. 保存断点和程序状态
    • 将程序的断点（返回地址）和被中断时的程序状态字寄存器PSWR的内容保存到栈或特定寄存器中（通常为支持嵌套而保存在栈中）
  3. 识别异常和中断并转到相应的处理程序

识别方式

软件识别方式

CPU设置一个异常状态寄存器，用于记录异常原因

  1. 操作系统使用一个统一的异常或中断查询程序，按优先级顺序查询异常状态寄存器，以检测异常和中断类型
  2. 先查询到的先被处理，然后转到内核中相应的处理程序

硬件识别方式（向量中断）

异常或中断处理程序的首地址称为中断向量

每个异常或中断都被指定一个中断类型号

在中断向量表中，类型号和中断向量一一对应

  1. CPU响应中断后，通过识别中断源获得中断类型号
  2. 据此计算出对应中断向量的地址
  3. 根据该地址从中断向量表中（一般形式为jmp 中断向量）取出中断服务程序的入口地址（即中断向量）
  4. 送入程序计数器PC，以转而执行中断服务程序

以上由中断隐指令（硬件自动）完成，以下由中断服务程序（OS）完成

  1. 保存现场和屏蔽字
    • 现场信息：用户可见的工作寄存器的内容
  2. 开中断
    • 允许更高级中断请求得到响应，实现中断嵌套
  3. 执行中断服务程序
  4. 关中断
    • 保证在恢复现场和屏蔽字时不被中断
  5. 恢复现场和屏蔽字
  6. 开中断
  7. 中断返回

多重中断和中断屏蔽技术

  • 单重中断
    • 若CPU在执行中断服务程序的过程中，又出现了新的更高优先级的中断请求，而CPU对新的中断请求不予响应
    • 无上述中断处理过程中的5和7
  • 多重中断（中断嵌套）
    • CPU暂停现行的中断服务程序，转去处理新的中断请求
  • CPU具备多重中断的功能需满足的条件
    • 在中断服务程序中提前设置开中断指令
    • 优先级别高的中断源有权中断优先级别低的中断源
  • 中断处理优先级：多重中断的实际优先级处理次序，可利用中断屏蔽技术动态调整
  • 中断屏蔽技术
    • 每个中断源都有一个屏蔽触发器
    • 1表示屏蔽该中断源的请求，0表示可以正常申请
    • 所有屏蔽触发器组合在一起构成屏蔽字寄存器，屏蔽字寄存器的内容称为屏蔽字
    • 可简单理解为屏蔽字中含1越多的中断源中断处理优先级越高

例如，中断处理优先级为A>B>C>D，则

B 0111
C 0011

caller调用callee的执行步骤
  1. caller将入口参数（实参）放在callee能访问到的地方
    • 调用参数通常集中存储在栈顶附近
  2. caller将返回地址存到特定的地方，然后将控制转移到callee
    • 由call指令实现
      • 将IP旧值（返回地址）入栈
        • 此时esp指向caller栈帧的最顶部（即将进入callee栈帧）
        • 栈帧最顶部一定是返回地址（当前函数的栈帧除外）
      • 设置IP新值，无条件转移到callee的第一条指令（进入callee栈帧）
  3. callee保存caller的现场（通用寄存器的内容）
    • 由enter指令实现
      • push ebp保存上一层函数的栈帧基址（ebp旧值）
      • mov ebp, esp（\((esp)\to ebp\)）设置当前函数的栈帧基址（ebp新值）
        • 此时，ebp、esp都指向callee的栈帧基址（内容是caller的栈帧基址）
        • 栈帧最底部一定是上一层栈帧的基址
  4. callee为其非静态局部变量分配空间
    • 局部变量通常集中存储在栈底附近
    • 即$[ebp]-4,[ebp]-8,\(（\)[ebp]$保存的是上一层栈帧的基址）
  5. 执行过程callee
    • 访问调用参数$[ebp]+8,[ebp]+12,\(（\)[ebp]+4$保存的是返回地址）
  6. callee将返回结果放到caller能访问到的地方
    • 一般放在eax寄存器
  7. callee恢复caller的现场，并释放局部变量所占空间
    • 由leave指令实现
      • mov esp, ebp（\((ebp)\to esp\)）使esp指向当前栈帧的底部
      • pop ebp将esp所指元素（内容为caller的基址）出栈（之后esp指向caller栈帧的顶部，内容为返回地址），写入ebp
  8. callee取出返回地址，将控制转移到caller
    • 由ret指令实现
      • 将函数的栈帧顶部找到IP旧值
      • 将其出栈并恢复IP寄存器

固态硬盘SSD

  • 原理：基于闪存技术Flash Memory，属于电可擦除ROM（\(E^2PROM\)）
  • 组成
    • 闪存翻译层：负责翻译逻辑块号，找到对应页
    • 存储介质：多个闪存芯片，每个芯片包含多个块，每个块包含多个页
  • 读写性能特性
    • 以页为单位读/写，相当于磁盘的扇区
    • 以块为单位“擦除”，擦干净的块，其中的每页都可以写一次，读无限次
    • 支持随机访问，系统给定一个逻辑地址，闪存翻译层可通过电路迅速定位到对应的物理地址
    • 读快、写慢，要写的页如果已经有数据，则不能写入，需要先将块内其他页全部复制到一个新的（擦除过的）块中，再写入新的页
  • 与机械硬盘相比
固态硬盘SSD 机械硬盘
读写速度快，随机访问性能高，用电路控制访问位置 通过移动磁臂旋转磁盘控制访问位置，有寻道时间和旋转延迟
安静无噪音，耐摔，能耗低，但造价更贵
一个块被擦除次数过多（重复写同一个块），可能会坏掉 机械硬盘的扇区不会因为写的次数太多而坏掉
  • 磨损均衡技术
    • 思想：将“擦除”平均分布在各个块上，以提升使用寿命
    • 动态磨损均衡：写入数据时，优先选择累计擦除次数少的新的闪存块
    • 静态磨损均衡：SSD检测并自动进行数据分配、迁移，让老旧的闪存块承担以读为主的存储任务，让较新的闪存块承担更多的写任务

操作系统

计算机系统概述

  • 操作系统是计算机系统中最基本的系统软件

  • 操作系统的特征

    • 并发
      • 两个或多个事件在同一时间间隔内发生
        • 并行：两个或多个事件在同一时刻发生
      • 宏观上多道，微观上交替运行，通过分时得以实现
    • 共享：系统中的资源可供内存中多个并发执行的进程共同使用
      • 互斥共享方式
        • 当且仅当某一进程访问完并释放资源（临界资源）后，才允许另一个进程对该资源进行访问
        • 临界资源：一段时间内只允许一个进程访问的资源
          • 例如，计算机中的大部分物理设备（如打印机）以及某些软件使用的栈、变量和表格等
      • 同时共享方式
        • 允许在一段时间内由多个进程分时交替访问
        • 例如，磁盘设备
    • 虚拟
      • 把一个物理上的实体变为若干逻辑上的对应物
        • 时分复用技术：如处理器的分时共享（虚拟处理器）
        • 空分复用技术：如虚拟存储器
    • 异步
      • 进程的执行不是一贯到底的，而是走走停停的，以不可预知的速度向前推进
      • 只要运行环境相同，OS需保持多次运行进程后都能获得相同的结果

  • 并发和共享是OS两个最基本的特征，两者互为存在的条件

    • 资源共享是以程序的并发为条件的，若系统不允许程序并发执行，则自然不存在资源共享问题
    • 若系统不能对资源共享实施有效的管理，则必将影响到程序的并发执行，甚至根本无法并发执行

  • OS的功能

    • 作为计算机系统资源的管理者

      • 处理机管理、存储器管理、文件管理、设备管理

    • 作为用户与计算机硬件系统之间的接口

      • 命令接口（直接使用OS服务） + 用户利用这些操作命令来组织和控制作业的执行
      • 分为联机命令接口和脱机命令接口
      • 程序接口（请求OS间接使用服务）
        • 由一组系统调用（广义指令）组成
        • 用户通过在程序中使用这些系统调用来请求OS为其提供服务

    • 实现了对计算机资源的扩充

    • 核心态（管态、内核态）：CPU可以执行特权指令

  • 多线程模型
    • 多对一模型：将多个用户级线程映射到一个内核级线程
      • 线程管理是在用户空间进行的，效率比较高
      • 如果一个线程在访问内核时发生阻塞，则整个进程都会被阻塞
      • 在任何时刻，只有一个线程能够访问内核，多个线程不能同时在多个处理机上运行
    • 一对一模型：将每个用户级线程映射到一个内核级线程
      • 当一个线程被阻塞后，允许调度另一个线程运行，所以并发能力较强
      • 每创建一个用户线程，相应地就需要创建一个内核线程，开销较大
    • 多对多模型：将n个用户级线程映射到m个内核级线程（\(n\ge m\)）

  • 调度的层次
    • 高级调度（作业调度）
      • 按照一定的原则从外存上处于后备队列的作业中挑选一个（或多个），给它（们）分配内存、输入/输出设备等必要的资源，并建立相应的进程，以使它（们）获得竞争处理机的权利
      • 内存与辅存之间的调度
      • 每个作业只调入一次、调出一次
    • 中级调度（内存调度）
      • 为了提高内存利用率和系统吞吐量
      • 将那些暂时不能运行的进程调至外存等待，此时进程的状态称为挂起态
      • 当它们己具备运行条件且内存又稍有空闲时，由中级调度来决定把外存上的那些己具备运行条件的就绪进程再重新调入内存，并修改其状态为就绪态，挂在就绪队列上等待
      • 实际上是存储器管理中的对换功能
    • 低级调度（进程调度）
      • 按照某种算法从就绪队列中选取一个进程，将处理机分配给它
      • 最基本的调度、各种OS必须配置、调度频率高

  • 调度的目标
    • \(\text{CPU利用率}=\frac{\text{CPU有效工作时间}}{\text{CPU有效工作时间}+\text{CPU空闲等待时间}}\)
    • 系统吞吐量：单位时间内CPU完成作业的数量
    • 周转时间
      • 从作业提交到作业完成所经历的时间
      • 是作业等待、在就绪队列中排队、在处理机上运行及输入/输出操作所花费的时间之和
      • \(\text{周转时间}=\text{作业完成时间}-\text{作业提交时间}\)
      • \(\text{平均周转时间}=\frac{1}{n}\sum\limits_{i=1}^{n} {\text{作业i的周转时间}}\)
      • \(\text{带权周转时间}=\frac{\text{作业周转时间}}{\text{作业实际运行时间}}\)
      • \(\text{平均带权周转时间}=\frac{1}{n}\sum\limits_{i=1}^{n} {\text{作业i的带权周转时间}}\)
    • 等待时间：进程处于等处理机的时间之和
    • 响应时间：从用户提交到系统首次产生响应所用的时间
  • 调度程序（调度器）：用于调度和分派CPU的组件
    • 排队器
    • 分派器
    • 上下文切换器
      • 将当前进程的上下文保存到其PCB中，再装入分派程序的上下文，以便分派程序运行
      • 移出分派程序的上下文，将新选进程的CPU现场信息装入处理机的各个相应寄存器

  • 不能进行进程的调度和切换的情况（应置系统的请求调度标志，直至该情况结束后才能进行调度和切换）
    • 在处理中断的过程
    • 进程在OS内核临界区中
    • 其他需要完全屏蔽中断的原子操作过程中
  • 应能进行进程调度与切换的情况
    • 发生引起调度条件且当前进程无法继续运行下去时
      • 若OS只在这种情况下进行调度，则是非剥夺调度
    • 中断处理结束或自陷处理结束后，返回被中断进程的用户态程序执行现场前，置请求调度标志
      • 若OS支持这种情况下进行调度，则是剥夺调度
  • 闲逛进程：若系统中没有就绪进程，则调度闲逛进程运行（优先级最低）
    • 若没有其他进程就绪，则该进程一直运行，并在执行过程中测试中断
    • 不需要CPU之外的资源，不会被阻塞
  • 典型的调度算法
    • 先来先服务FCFS调度算法
      • 每次从后备作业队列中选择最先进入该队列的作业调入内存
      • 每次从就绪队列中选择最先进入该队列的进程，将处理机分配给它，使其投入运行
    • 短作业优先SJF调度算法
      • 每次从后备队列中选择一个或若干个估计运行时间最短的作业，调入内存
      • 每次从就绪队列中选择一个估计运行时间最短的进程，将处理机分配给它，使其投入运行
    • 高响应比优先HRRN调度算法
      • 先计算后备队列中每个作业的响应比，从中选出响应比最高的作业投入内存
      • \(\text{响应比}R_p=\frac{\text{等待时间}+\text{要求服务时间}}{\text{要求服务时间}}\)
      • 综合考虑了作业的等待时间和估计运行时间
    • 优先级调度算法
      • 每次从后备作业队列中选择优先级最高的一个或多个作业，调入内存
      • 每次从就绪队列中选择优先级最高的进程，将处理机分配给它，使其投入运行
    • 时间片轮转调度算法
      • 所有就绪进程按先来先服务排队，总是选择就绪队列中第一个进程运行一个时间片，然后释放处理机给下一个就绪进程
      • 被剥夺的进程返回到就绪队列的末尾继续排队（若没有运行完毕）
    • 多级反馈队列调度算法
      • 设置多个就绪队列，优先级越高的队列进程的运行时间片越小，优先级按\(1\sim n\)级队列递减
      • 当一个新进程进入内存后，首先将其放入第1级队列的末尾，按FCFS原则排队等待调度
      • 当轮到位于第i级队列的某一进程执行时
        • 若它在对应时间片内完成，则可撤离系统
        • 否则，将该进程转入下一级队列的末尾（第n级队列采用时间片轮转方式运行）
      • 仅当第\(1\sim (i-1)\)级队列均为空时，才会调度第i级队列中的进程运行
      • 第i级队列中的进程运行时，若又有新进程进入第\(1\sim (i-1)\)级中的任一队列，则把正在运行的进程放回第i级队列的末尾，把处理器分配给新进程

  • 临界资源：一次仅允许一个进程使用的资源
    • 大多数物理设备，如打印机等；可以被若干进程共享的变量、数据等
    • 对临界资源的访问，必须互斥地进行
    • 临界资源的访问过程
      • 进入区
        • 检查是否可以进入临界区
        • 若能进入则设置正在访问临界区的标志，以阻止其他进程同时进入临界区
      • 临界区：进程中访问临界资源的代码
      • 退出区：将正在访问临界区的标志清除
      • 剩余区：代码中的剩余部分
  • 同步（直接制约关系）：即进程之间有先后顺序
  • 互斥（间接制约关系）
    • 当一个进程进入临界区使用临界资源时，另一个进程必须等待
    • 当占用临界资源的进程退出临界区后，另一进程才允许去访问此临界资源
  • 为禁止两个进程同时进入临界区，同步机制应遵循以下准则
    • 空闲让进：临界区空闲时，可以允许一个请求进入临界区的进程立即进入临界区
    • 忙则等待：当己有进程进入临界区时，其他试图进入临界区的进程必须等待
    • 有限等待：对请求访问的进程，应保证能在有限时间内进入临界区
    • 让权等待：当进程不能进入临界区时，应立即释放处理器，防止进程忙等待

  • 单标志法：确保进程互斥访问临界区，两个进程必须交替进入临界区，违背空闲让进
  • 双标志先检查法：进程不必交替进入、两进程可能同时进入临界区，无法实现互斥访问，违背忙则等待
  • 双标志后检查法：确保进程互斥访问临界区、存在两个进程都进入不了临界区的“饥饿”现象，违背空闲让进、有限等待
  • Peterson's Algorithm：确保进程互斥访问临界区、不会出现”饥饿“现象、违背让权等待

  • 互斥锁

    • 一个进程在进入临界区时通过函数acquire()获得锁
    • 在退出临界区时通过函数release()释放锁
    • 每个互斥锁有一个布尔变量available，表示锁是否可用
      • 若可用，则调用acquire()会成功，且锁不再可用
    • 当一个进程试图获取不可用的锁时，会被阻塞，直到锁被释放
    • acquire()和release()的执行必须是原子操作，因此互斥锁通常采用硬件机制来实现
    • 缺点：忙等待，通常用于多处理器系统（一个进程独占一个处理机等待，其他进程不受影响）
  • 信号量
    • 一种功能较强的机制
    • 只能被两个标准的原语wait(S)和signal(S)访问，也可记为P操作和V操作
  • 整型信号量
    • 定义一个用于表示资源数目的整型量S
    • 不遵循“让权等待”原则
  • 记录型信号量
    • 定义一个用于代表资源数目的整型变量value和一个进程链表L，用于链接所有等待该资源的进程
  • 利用信号量实现同步
    • 初始化一对同步进程的公共信号量S=0
    • 前VP后，即在前操作之后V(S)，在后操作之前P(S)
  • 利用信号量实现互斥
    • 初始化一对互斥进程的信号量S=1（即可用资源数为1）
    • 把临界区置于P(S)和V(S)之间
  • 利用信号量实现前驱关系（多级同步）
    • 为每一对前驱关系的进程设置一个信号量
    • 初始化信号量均为0
    • 前VP后，即在前操作之后V(S)，在后操作之前P(S)
  • 管程
    • 组成：管程的名称、局部于管程内部的共享数据结构说明、对该数据结构进行操作的一组过程（或函数）、对局部于管程内部的共享数据设置初始值的语句
    • 管程把对共享资源的操作封装起来，管程内的共享数据结构只能被管程内的过程所访问
    • 一个进程只有通过调用管程内的过程才能进入管程访问共享资源
    • 每次仅允许一个进程进入管程，从而实现进程互斥
  • 条件变量（阻塞的原因）
    • 每个条件变量保存了一个等待队列，用于记录因该条件变量而阻塞的所有进程
    • 对条件变量只能进行两种操作，即wait和signal
      • x.wait：当x对应的条件不满足时，正在调用管程的进程调用x,wait将自己插入x条件的等待队列，并释放管程
      • x.signal：x对应的条件发生了变化，则调用x.signal，唤醒一个因x条件而阻塞的进程
  • 死锁：多个进程因竞争资源而造成的一种僵局（互相等待，若无外力作用，这些进程都将无法向前推进）
  • 死锁产生的原因：系统资源的竞争、进程推进顺序非法
  • 死锁产生的必要条件
    • 互斥条件
      • 进程要求对所分配的资源(如打印机)进行排他性使用
      • 即在一段时间内某资源仅为一个进程所占有
    • 不剥夺条件
      • 进程所获得的资源在未使用完之前，不能被其他进程强行夺走
      • 即只能由获得该资源的进程自己来释放（只能是主动释放）
    • 请求并保持条件
      • 进程己经保持了至少一个资源，但又提出了新的资源请求
      • 而该资源已被其他进程占有，此时请求进程被阻塞
      • 但对自己已获得的资源保持不放
    • 循环等待条件
      • 存在一种进程资源的循环等待链，链中每个进程已获得的资源同时被链中下一个进程所请求
  • 死锁的处理策略
    • 死锁预防：设置某些限制条件，破坏产生死锁的4个必要条件中的一个或几个
    • 避免死锁：在资源的动态分配过程中，用某种方法防止系统进入不安全状态
    • 死锁的检测及解除：允许进程在运行过程中发生死锁。通过系统的检测机构及时地检测出死锁的发生，然后采取某种措施解除死锁
资源分配策略 各种可能模式 主要优点 主要缺点
死锁预防 保守，宁可资源闲置 一次请求所有资源，资源剥夺，资源按序分配 适用于突发式处理的进程，不必进行剥夺 效率低、剥夺次数过多、不便灵活申请新资源
死锁避免 折中（在运行时判断是否可能死锁） 寻找可能得安全允许顺序 不必进行剥夺 必须知道将来的资源需求、进程不能被长时间阻塞
死锁解除 宽松，只要允许就分配资源 定期检查死锁是否已经发生 不延长进程初始化时间，允许对死锁进行现场处理 通过剥夺解除死锁，造成损失
  • 死锁预防
    • 破坏互斥条件
      • 允许系统资源都能共享使用
      • 有些不能做到，而且有些场合需要保持互斥性
    • 破坏不剥夺条件
      • 请求新的资源而得不到满足时，必须释放已经保持的所有资源
      • 可能造成前一阶段工作的失效
      • 常用于状态易于保存和恢复的资源
    • 破坏请求并保持条件
      • 预先静态分配
      • 进程在运行前一次申请完它所需要的全部资源，在它的资源未满足前，不把它投入运行
      • 一旦投入运行，这些资源就一直归它所有，不再提出其他资源请求
    • 破坏循环等待条件
      • 顺序资源分配法
      • 给系统中的资源编号，规定每个进程必须按编号递增的顺序请求资源，同类资源一次申请完
      • 编号必须相对稳定，这就限制了新类型设备的增加
  • 安全状态：系统能按某种进程推进顺序（安全序列）为每个进程分配其所需的资源，直至满足每个进程对资源的最大需求，使每个进程都可顺序完成
  • 并非所有的不安全状态都是死锁状态
    • 当系统进入不安全状态时，便可能进入死锁状态
    • 只要系统处于安全状态，便可避免进入死锁状态
  • 资源分配图
    • 圆圈表示进程，框代表一类资源，框中的一个圆代表一类资源中的一个资源
    • 请求边：从进程到资源的有向边，表示该进程申请一个单位的该类资源
    • 分配边：从资源到进程的有向边，表示该类资源已有一个资源分配给了该进程
    • 简化资源分配图可检测系统状态S是否为死锁状态

  • 资源分配图的简化方法
    • 寻找一个所有请求边都可以被满足的进程
      • 若寻找成功，则将该进程及相连的所有边（包括请求边和分配边）删除
    • 若能删除图中所有的边，则称该图是可完全简化的
    • 若寻找失败且还有边没有被删除，则处于死锁状态
  • 死锁定理：S为死锁的条件是当且仅当S状态的资源分配图是不可完全简化的
  • 死锁解除
    • 资源剥夺法
      • 挂起某些进程，抢占其资源，将这些进程分配给其他的死锁进程
      • 应防止被挂起的进程长时间得不到资源而处于资源匮乏的状态
    • 撤销进程法
      • 强制撤销部分甚至全部死锁进程并剥夺这些进程的资源
      • 撤销的原则可以按进程优先级和撤销进程代价的高低进行
    • 进程回退法
      • 让一（或多）个进程回退到足以回避死锁的地步，进程回退时自愿释放资源而非被剥夺
      • 要求系统保持进程的历史信息，设置还原点

内存管理

  • 内存的管理的主要功能
    • 内存空间的分配与回收：OS完成主存空间的分配与管理
    • 地址转换：将逻辑地址转换为物理地址
    • 内存空间的扩充：利用虚拟技术从逻辑上扩充内存
    • 内存共享：允许多个进程访问内存的同一部分
    • 存储保护：保证各道作业在各自的存储空间内运行，互不干扰
  • 用户源程序到可在内存中执行的程序
    • 编译：由编译程序将用户源代码编译成若干目标模块（每个目标模块都是从0开始的相对地址）
    • 链接：由链接程序将编译后形成的一组目标模块及它们所需的库函数链接在一起，形成一个完整的装入模块
    • 装入：由装入程序将装入模块装入内存运行

  • 程序的链接方式

    • 静态链接：链接为一个完整的装入模块，以后不再拆开、需修改相对地址、需变换外部调用符号
    • 装入时动态链接：边装入边链接、便于修改和更新、便于实现对目标模块的共享
    • 运行时动态链接：在程序执行中需要该目标模块时才将该目标模块调入内存并链接到装入模块上、加快程序的装入过程、可节省大量内存空间

  • 装入方式

    • 绝对装入
      • 按照装入模块中的地址，将程序和数据装入内存
      • 只适用于单道程序环境
      • 程序中的逻辑地址与实际内存地址完全相同
    • 可重定位装入
      • 根据内存的当前情况，将装入模块装入内存的适当位置
      • 在进程装入时完成地址变换（静态重定位）
    • 动态运行时装入
      • 装入程序把装入模块装入内存后，并不立即把装入模块中的相对地址转换为绝对地址，而是把这种地址转换推迟到程序真正要执行时才进行
      • 需要一个重定位寄存器的支持
      • 可以把程序分配到不连续的存储区
      • 在程序运行之前只装入部分代码即可投入运行
      • 在程序运行期间，可以根据需要动态申请分配内存
      • 便于程序段的共享

  • 逻辑地址与物理地址

    • 编译后，每个目标模块都从0号单元开始编址，这称为该目标模块的相对地址（逻辑地址）
    • 当链接程序将各个模块链接成一个完整的可执行目标程序时，链接程序顺序依次按各个模块的相对地址构成统一的从0号单元开始编址的逻辑地址空间（或虚拟地址空间）
      • 对于32位系统，逻辑地址空间的范围是\(0\sim 2^{32}-1\)
    • 进程在运行时，看到和使用的地址都是逻辑地址
      • 内存管理的具体机制对用户透明
    • 物理地址空间是指内存中物理单元的集合，是地址转换的最终地址，进程在运行时执行指令和访问数据，最后都要通过物理地址从主存中存取
    • 当装入程序将可执行代码装入内存时，必须通过地址转换将逻辑地址转换成物理地址，这个过程称为地址重定位
    • 操作系统通过内存管理部件（MMU） 将进程使用的逻辑地址转换为物理地址
    • 进程使用虚拟内存空间中的地址，操作系统在相关硬件的协助下，将它"转换”成真正的物理地址
    • 逻辑地址通过页表映射到物理内存，页表由操作系统维护并被处理器引用

  • 进程的内存映像

    • 代码段：只读、可被多个进程共享
    • 数据段：包括全局变量和静态变量
    • 进程控制块
    • 堆：用于存放动态分配的变量，通过调用malloc函数动态地向高地址分配空间
    • 栈：用于实现函数调用，从用户空间的最大地址向低地址方向增长

  • 内存保护

    • 在CPU中设置一对上、下限寄存器，存放用户作业在主存中的下限和上限地址
    • 采用重定位寄存器（又称基地址寄存器）和界地址寄存器（又称限长寄存器）来实现
      • 重定位寄存器存储最小的物理地址
      • 界地址寄存器存储最大的逻辑地址
      • 若逻辑地址小于等于界地址寄存器的值，则加上重定位寄存器的值即为物理地址，否则发生地址越界
      • 加载重定位寄存器和界地址寄存器时必须使用特权指令，因此只允许OS内核修改
  • 并不是所有的进程内存空间都适合共享，只有那些只读的区域才可以共享
  • 可重入代码（又称纯代码）：一种允许多个进程同时访问但不允许被任何进程修改的代码

  • 内存分配

    • 连续分配：一次性分配完整的连续空间
    • 离散分配：一次性分配完整的可分散空间
    • 请求分配：运行时不需要把作业的所有页面都装入内存

  • 单一连续分配

    • 将内存划分为系统区和用户区
      • 系统区仅供OS使用，通常位于低地址部分
      • 用户区中仅有一道用户程序
    • 简单、无外部碎片、无须进行内存保护
    • 只能用于单用户、单任务的OS、有内部碎片、存储器的利用率极低

  • 固定分区分配

    • 将用户内存空间划分为若干固定大小的区域，每个分区只装入一道作业
    • 划分分区的方法
      • 分区大小相等
      • 分区大小不等：划分为多个较小的分区、适量的中等分区和少量大分区
    • 建立一张分区使用表，每个表项包括该分区的起始地址、大小以及状态（是否已分配）
    • 程序太大而不能放入任何一个分区时，需使用覆盖技术
    • 程序小于最小分区时，分区内部存在空间浪费，即内部碎片
    • 无外部碎片、不能实现多进程共享一个主存区，存储空间利用率低

  • 动态分区分配（可变分区分配）

    • 在进程装入内存时，根据进程的实际需要，动态为其分配内存，使得分区的大小正好适合进程的需要
    • 系统中分区的大小和数目是可变的
    • 随着时间的推移，内存中会产生较多的外部碎片，可采用紧凑技术解决、无内部碎片

  • 动态分区的分配策略

算法名称 空闲分区链接方式 分配内存 特点
首次适应算法 以地址递增的次序链接 从链首开始顺序查找第一个满足要求的空闲分区 简单、查找速度快、平均性能最好、碎片多出现于低地址空间
邻近适应算法（循环首次适应算法） 以地址递增的次序链接 从上次查找结束的位置开始继续查找 平均性能比首次适应差、碎片多出现于高地址空间
最佳适应算法 以容量递增的次序链接 从链首开始顺序查找第一个满足要求（且最小）的空闲分区 排序开销大、形成许多难以利用的小碎片、性能通常很差
最坏适应算法 以容量递减的次序链接 从链首开始顺序查找第一个满足要求（且最大）的空闲分区 排序开销大、使系统缺少大的连续空闲地址空间
  • 动态分区回收内存空间时需要与相邻的空闲分区进行合并（如果有的话）
  • 分页
    • 把主存空间划分为大小相等且固定的块，块相对较小，作为主存的基本单位
    • 每个进程也以块为单位进行划分
    • 进程在执行时，以块为单位逐个申请主存中的块空间
  • 每个进程平均只产生半个块大小的内部碎片
页（页面） 进程中的块
页框（页帧） 内存中的块
块（盘块） 外存中的块

  • 分页存储管理的逻辑地址结构：页号P 页内偏移量W

  • 地址结构决定了虚拟内存的寻址空间

  • 硬件支持：页表寄存器PTR，存放页表在内存的起始地址F和页表长度M

  • 系统为每个进程都建立一张页表，记录页面在内存中对应的物理块号

    • 页表项：页号（从0开始，可隐含）+（物理）块号

  • 地址变换机构

    1. 将页号P与页表长度M比较
      • 若\(P\ge M\)，则表示地址越界并中断
      • 若\(P<M\)，则将页表起始地址F与（页号P$$页表项长度）相加，得到该表项在页表中的位置，从中得到该页的物理块号，装入物理寄存器中
    2. 将逻辑地址中的页内偏移量送入物理地址寄存器中的块内地址字段，即可得到物理地址物理块号 页内偏移量

  • 快表
    • 具有并行查找能力的高速缓冲存储器，又称相联存储器
    • 用于存放当前访问的若干页表项
  • 在具有快表的分页机制中，地址变换过程如下
    1. CPU给出逻辑地址，由硬件进行地址转换，将页号送入高速缓存寄存器，并将此页号与快表中的所有页号进行比较
    2. 若找到匹配的页号，说明所要访问的页表项在快表中，则直接从中取出该页对应的页框号，与页内偏移量拼接形成物理地址
    3. 若未找到匹配的页号，则需要访问主存中的页表，读出页表项后，应同时将其存入快表，以便之后可能再次访问
      • 若快表已满，则需按特定的算法淘汰一个旧页表项
  • 有些处理器设计为快表和同时查找，若快表查找成功则终止慢表的查找

  • 两级页表
    • 顶级页表最多只能有1个页面
    • 逻辑地址空间的格式：页目录项 页表项 页内偏移
  • 基本分段存储管理
    • 逻辑地址结构：段号S 段内偏移量W
  • 段表
    • 每个进程都有一张逻辑空间与内存空间映射的段表
    • 每个段表项对应进程的一段
    • 段表项：段号 段长 本段在内存的始址
  • 地址变换机构
    1. 比较段号S和段表长度M，若\(S\ge M\)，则产生越界中断，否则继续执行
    2. 段表中段号S对应的段表项地址=段表始址F+段号S$$段表项长度
    3. 取出该段表项中的段长C，若段内偏移量\(\ge C\)，则产生越界中断，否则继续执行
    4. 取出该段表项中的始址b，物理地址即为\(E=b+W\)

  • 段的共享是通过两个作业的段表中相应表项执行被共享的段的同一个物理副本来实现的
  • 段页式管理
    • 逻辑地址结构：段号 页号 页内偏移量
    • 在一个进程中，段表只有一个，而页表可能有多个

  • 传统存储管理方式的特征

    • 一次性：作业需一次性全部装入内存后才能开始运行
    • 驻留性：作业被装入内存后，就一直驻留在内存中，直至运行结束

  • 虚拟存储器的主要特征

    • 多次性：作业允许被分成多次调入内存运行，只需将当前运行的部分程序和数据调入内存即可开始运行
    • 对换性：在进程运行期间，允许将暂不使用的程序和数据从内存调至外存的对换区（换出）
    • 虚拟性：从逻辑上扩充内存的容量

  • 虚拟内存的实现方式：请求分页存储管理、请求分段存储管理、请求段页式存储管理

  • 虚拟内存的实现需要的支持

    • 一定容量的内存和外存
    • 页表机制（或段表机制）：作为主要的数据结构
    • 中断机构：当用户程序要访问的部分尚未调入内存时，则产生中断
    • 地址变换机构：逻辑地址到物理地址的变换

  • 请求分页系统中页表的表项

    • 页号
    • 状态位P：=1表示该页是否已调入内存，供程序访问时参考
    • 访问字段A：用于记录本页在一段时间内被访问的次数，或记录本页最近已有多长时间未被访问，供置换算法换出页面时参考
    • 修改位M：标识该页在调入内存后是否被修改过，以确定页面置换时是否写回外存
    • 外存地址：用于指出该页在外存上的地址，通常是物理块号，供调入该页时参考

  • 缺页中断机构

    • 当所要访问的页面不在内存中时，便产生一个缺页中断，请求OS将所缺的页调入内存
    • 应将缺页的进程阻塞（调页完成唤醒）
      • 若内存中有空闲块，则分配一个块，将要调入的页装入该块，并修改页表中的相应页表项
      • 若此时内存中没有空闲块，则要淘汰某页（若被淘汰的页的修改位=1，则要将其写回外存）
    • 与一般中断的区别
      • 在指令执行期间而非一条指令执行完成后产生和处理中断信号，属于内部异常
      • 一条指令在执行期间，可能产生多次缺页中断

  • 地址变换机构

    • 先检查快表
    • 若找到要访问的页，则修改页表项中的访问位（写指令还需要置修改位），然后利用页表项中的物理块号与页内地址形成物理地址物理块号 页内地址
    • 若未找到该页的页表项，则应到内存中查找页表
      • 若页面已调入内存，则将该页的页表写入快表
        • 若快表已满，则需采用某种算法替换
      • 若页面未调入，则产生缺页中断，请求从外存把该页调入内存

  • 驻留集：给一个进程分配的物理页框的集合

    • 分配给一个进程的页框越少，驻留在内存中的进程就越多，从而可提高CPU的利用率
    • 若一个进程在主存中的页面过少，则尽管有局部性原理，缺页率仍相对较高
    • 若分配的页框过多，则由于局部性原理，对该进行的缺页率没有太明显的影响

  • 内存分配策略

    • 固定分配局部置换
      • 固定分配：进程运行期间不改变驻留集大小
      • 局部置换：若发生缺页则在已分配的页面中选择一页换出，保证分配给该进程的内存空间不变
    • 可变分配全局置换
      • 可变分配：进程运行期间可根据情况适当增加或减少驻留集大小
      • 全局置换：若发生缺页，则系统从空闲物理块队列中取出一块分配给该进程，并将所缺页调入
    • 可变分配局部置换
      • 频繁发生缺页中断，则适当再分配
      • 缺页率较低，则适当减少分配

  • <固定/可变>分配：指进程运行期间是否改变驻留集的大小

  • <全局/局部>置换：指缺页时是否增加驻留集的大小（因此固定分配只能对应局部置换）

  • 物理块调入算法

    • 平均分配算法：将系统中所有可供分配的物理块平均分配给各个进程
    • 按比例分配算法：根据进程的大小按比例分配
    • 优先权分配算法：为重要和紧迫的进程分配较多的物理块

  • 调入页面的时机

    • 预调页策略：将预计在不久之后便会被访问的页面预先调入内存
    • 请求调页策略：缺一页调一页

  • 请求分页系统中外存分为两部分

    • 用于存放文件的文件区：采用离散分配方式
    • 用于存放对换页面的对换区：采用连续分配方式、磁盘I/O速度较快

  • 从何处调入页面

    • 系统拥有足够的对换区空间
      • 可以全部从对换区调入所需页面
      • 在进程运行前，将与该进程有关的文件从文件区复制到对换区
    • 系统缺少足够的对换区空间
      • 不会被修改的文件直接从文件区调入
      • 可能被修改的部分换出时调到对换区，之后需要时再从对换区调入
    • UNIX方式
      • 未运行过的页面都从文件区调入
      • 曾经运行过但又被换出的页面，下次调入时从对换区调入
      • 共享页面若被其他进程调入内存，则无须从对换区调入

  • 页面置换算法

  • 最佳OPT置换算法

    • 被淘汰的页面是以后永不使用的页面，或是最长时间内不再被访问的页面
    • 无法实现

  • 先进先出FIFO页面置换算法

    • 优先淘汰最早进入内存的页面
    • 会产生所分配的物理块数增大而缺页率不减反增的异常现象（Belady异常）（仅FIFO会出现Belady异常）

  • 最近最久未使用LRU置换算法

    • 淘汰最近最长时间未访问过的页面
    • 性能较好、堆栈类算法

  • 简单时钟CLOCK置换算法（最近未用NRU算法）

    • 首次被装入或被访问时，置访问位为1
    • 某页被替换时，替换指针指向被替换页面的下一页
    • 选择访问位为0的页面替换
    • 若为1，则将其置为0，暂不换出（给予该页第二次驻留内存的机会），再检查下一个页面

  • 改进型CLOCK置换算法四类页面（访问位A、修改位M）

1类 A=0,M=0 最近未被访问且未被修改 最佳淘汰页
2类 A=0,M=1 最近未被访问但已被修改
3类 A=1,M=0 最近已被访问但未被修改 可能再被访问
4类 A=1,M=1 最近已被访问且已被修改 可能再被访问
  • 改进型CLOCK算法执行过程
    1. 从指针的当前位置开始，扫描循环队列，寻找A=0且M=0的1类页面，将遇到的第一个1类页面作为选中的淘汰页（第一次扫描不改变A）
    2. 若第1步失败，则进行第2次扫描：寻找A=0且M=1的2类页面，将遇到的第一个2类页面作为淘汰页（第二轮扫描置A=0）
    3. 若第2步失败，则将指针返回到开始的位置，并将所有帧的A置0，重复第一步，若有必要，重复第二步，此时一定能找到被淘汰的页
  • 抖动
    • 频繁的页面调度行为
    • 根本原因：同时运行的进程太多、分配给每个进程的物理块太少
  • 工作集
    • 某段时间间隔内，进程要访问的页面集合
    • 反映了进程在接下来的一段时间内很有可能会频繁访问的页面集合
    • 若分配给进程的物理块小于工作集大小，则该进程就很有可能频繁缺页
    • 因此分配给进程的物理块数（即驻留集大小）要大于工作集大小
    • t时刻的工作集即为t时刻及之前的<工作集>大小个页面的集合
  • 例如（工作集大小为5）
    • \(t_1\)时刻的工作集为\(\{2,3,5\}\)，\(t_2\)时刻的工作集为\(\{1,2,3,4\}\)
  • 内存映射文件
    • 将磁盘文件的全部或部分内容与进程虚拟地址空间的某个区域建立映射关系
    • 可直接访问被映射的文件，而不必执行文件I/O操作，也无须对文件内容进行缓存处理
    • 磁盘的周期性分页是由OS在后台隐蔽实现的，对应用程序员完全透明
  • 虚存的容量
    • 虚存的实际容量\(\le\)内存容量+外存容量
    • 虚存的最大容量\(\le\)计算机的地址位数所能容纳的最大容量

文件管理

  • 在系统运行时，计算机以进程为基本单位进行资源的调度和分配
  • 在用户进行的输入、输出中，以文件为基本单位
  • 文件的属性：名称、类型、创建者、所有者、位置、大小、保护、创建时间、最后一次修改时间和最后一次存取时间
  • OS通过文件控制块FCB来维护文件元数据
  • 文件控制块FCB是用来存放控制文件需要的各种信息的数据结构，以实现按名存取
  • FCB的有序集合称为文件目录
  • 创建一个新文件，OS将分配一个FCB并存放在文件目录中，称为目录项
  • FCB主要包含：文件基本信息、存取控制信息、使用信息
  • 一个文件目录也被视为一个文件，称为目录文件
  • 索引结点
    • 将文件名和文件描述信息分开，文件描述信息单独形成一个称为索引结点的数据结构，简称i结点（inode）
    • 文件目录中的每个目录项仅由文件名和指向该文件所对应的i结点的指针构成
    • 磁盘索引结点
      • 存放在磁盘上的索引结点
      • 每个文件有一个唯一的磁盘索引结点
      • 主要包括：文件标识符、文件类型、文件存取权限、文件物理地址、文件长度、文件链接计数、文件存取时间
    • 内存索引结点
      • 存放在内存中的索引结点
      • 当文件被打开时，要将磁盘索引结点复制到内存中，便于后续使用
      • 增加了：索引结点编号、状态、访问计数、逻辑设备号、链接指针
  • 文件的基本操作：创建文件（分配必要的外存空间、创建目录项）、写文件、读文件、重新定位文件、删除文件、截断文件
  • 文件的打开与关闭
    • 文件首次使用之前通过系统调用open被显式地打开
    • OS维护一个打开文件表，将文件的信息从外存复制到打开文件表的一个表目中
    • OS将该表目的编号（也称索引）返回给用户
    • 当用户再次向系统发出文件操作请求时，可通过索引在打开文件表中查找文件信息
    • 当文件不再使用时，可利用close系统调用关闭，OS删除对应表目
    • 整个系统的打开文件表：FCB的副本及其他信息
    • 每个进程的打开文件表：根据其打开的文件，指向系统表中的对应条目
    • 系统打开文件表为每个文件关联一个打开计数器，以记录打开该文件的进程数目
      • 归零时，才可从系统打开文件表中删除相应条目
    • 每个打开文件都具有如下关联信息：文件指针、文件打开计数、文件磁盘位置、访问权限
  • 文件保护：口令保护、加密保护、访问控制
  • 访问控制
    • 为每个文件和目录增加一个访问控制列表ACL，以规定每个用户名及其所允许的访问类型
    • 精简的访问列表：拥有者、组、其他

输入/输出管理

  • 设备的分类
    • 按信息交换的单位
      • 块设备（信息交换以数据块为单位）
        • 属于有结构设备、传输速率高、可寻址（即可随机读写）
        • 如磁盘
      • 字符设备（信息交换以字符为单位）
        • 属于无结构类型、传输速率低、不可寻址、时常采用中断I/O方式
        • 如交互式中断机、打印机等
    • 按传输速率：低速设备、中速设备、高速设备
  • I/O接口（设备控制器）
    • 组成
      • 设备控制器与CPU的接口
        • 数据线与数据寄存器和控制/状态寄存器相连
        • 地址线、控制线与I/O逻辑相连
      • 设备控制器与设备的接口
        • 一个设备控制器可以连接一个或多个设备，因此控制器中有一个或多个设备接口
        • 每个接口中都存在数据、控制和状态三种类型的信号
      • I/O逻辑
        • 用于实现对设备的控制
        • 对从CPU收到的I/O命令进行译码
        • 通过收到的地址对所选设备进行控制
    • 功能：接收和识别CPU发来的命令、数据交换、标识和报告设备的状态，以供CPU处理、地址识别、数据缓冲、差错控制
  • I/O端口：设备控制器中可被CPU直接访问的寄存器
    • 数据寄存器：存放从设备送来的输入数据或从CPU送来的输出数据
    • 状态寄存器：存放设备的状态信息
    • 控制寄存器：存放从CPU送来的控制信息
  • I/O端口的编址
    • 独立编址：普通用户程序不能对其进行访问，只有OS使用特殊的I/O指令才能访问端口
    • 统一编址：内存映射I/O、通常分配高地址区域
  • 接口的分类：字符设备接口（通常采用中断驱动方式）、块设备接口（如磁盘，通常采用DMA方式）、网络设备接口、阻塞/非阻塞I/O接口
    • 阻塞I/O接口：当用户进程调用I/O操作时，进程就被阻塞，需要等待I/O操作完成，进程才被唤醒继续执行
    • 非阻塞I/O接口：用户进程调用I/O操作时，不阻塞该进程，该I/O调用返回一个错误返回值，通常，进程需要通过轮询的方式来查询I/O操作是否完成
  • I/O控制方式
    • 程序直接控制方式
    • 中断驱动方式
    • DMA方式
      • 在I/O设备和内存之间开辟直接的数据交换通路
      • 基本单位是块
      • 仅在传送一个或多个数据块的开始和结束时，才需CPU干预
      • 整块数据的传送是在DMA控制器的控制下完成的
    • 通道控制方式：基本传送单位是一组数据块、可实现CPU、通道和I/O设备三者的并行操作
  • DMA控制器中设置的4类寄存器
    • 命令/状态寄存器IR：接收从CPU发来的I/O命令、有关控制信息，或设备的状态
    • 内存地址寄存器MAR
      • 在输入时，存放把数据从设备传送到内存的起始目标地址
      • 在输出时，存放由内存到设备的内存源地址
    • 数据寄存器DR：暂存从设备到内存或从内存到设备的数据
    • 数据计数器DC：存放本次要传送的字（节）数
  • 设备独立性（设备无关性）
    • 使得应用程序独立于具体使用的物理设备
    • 在应用程序中，使用逻辑设备名来请求使用某类设备
    • 在系统实际执行时，必须将逻辑设备名映射成物理设备名使用
    • 使用逻辑设备名的好处
      • 增加设备分配的灵活性
      • 易于实现I/O重定向（用于I/O操作的设备可以更换，而不必改变应用程序）
  • I/O软件层次结构
    • 用户层I/O软件
      • 实现与用户交互的接口
      • 用户可直接调用在用户层提供的、与I/O操作相关的库函数，对设备进行操作
      • 用户层软件必须通过一组系统调用来获取操作系统服务
    • 设备独立性软件
      • 用于实现用户程序与设备驱动器的统一接口、设备命令、设备的保护及设备的分配与释放等，同时为设备管理和数据传送提供必要的存储空间
      • 执行所有设备的公共操作
        • 对设备的分配与回收
        • 将逻辑设备名映射为物理设备名
        • 对设备进行保护，禁止用户直接访问设备
        • 缓冲管理、差错控制
        • 提供独立于设备的大小统一的逻辑块，屏蔽设备之间信息交换单位大小和传输速率的差异
      • 向用户层提供统一接口
    • 设备驱动程序
      • 与硬件直接相关，负责具体实现系统对设备发出的操作指令，驱动I/O设备工作的驱动程序
      • 通常，每类设备配置一个设备驱动程序，它是I/O进程与设备控制器之间的通信程序
      • 通常以进程的形式存在
      • 设备驱动程序向上层用户程序提供一组标准接口，设备具体的差别被设备驱动程 序所封装，用于接收上层软件发来的抽象I/O要求（如read和write命令），转换为具体要求后，发送给设备控制器，控制I/O设备工作
      • 将由设备控制器发来的信号传送给上层软件，从而为I/O内核子系统隐藏设备控制器之间的差异
    • 中断处理程序
  • 磁盘高速缓存
    • 利用内存中的存储空间来暂存从磁盘中读出的一系列盘块中的信息
    • 逻辑上属于磁盘，物理上则是驻留在内存中的盘块
  • 缓冲区
    • 引入目的：缓和CPU与I/O设备间速度不匹配的矛盾、减少对CPU的中断频率，放宽对CPU中断响应时间的限制、解决基本数据单元大小（即数据粒度）不匹配的问题、提高CPU和I/O设备之间的并行性
    • 实现方法：硬件缓冲区、位于内存区域的缓冲区
    • 缓冲技术分类：单缓冲、双缓冲、循环缓冲、缓冲池
  • 磁盘把一块数据输入到缓冲区的时间为T、OS将该缓冲区中的数据传送到用户区的时间为M、CPU对该数据块的处理时间为C
  • 单缓冲：在主存中设置一个缓冲区、处理每块数据的用时为\(\max\{C,T\}+M\)
  • 双缓冲：提高了处理机和输入设备的并行程度、处理每块数据的用时为\(\max\{M+C,T\}\)
    • 若\(M+C<T\)，则块设备可连续输入
    • 若\(M+C>T\)，则CPU不必等待设备输入
单缓冲 双缓冲
  • 为实现双向数据传输，必须在两台机器中都设置两个缓冲区，一个用作发送缓冲区，另一个用作接收缓冲区
  • 循环缓冲
    • 包含多个大小相等的缓冲区，通过链接指针构成一个环形
    • \(in\)指针指向可以输入数据的第一个空缓冲区
    • \(out\)指针指向可以提取数据的第一个满缓冲区

  • 缓冲池
    • 三个队列：空缓冲队列、装满输入数据的缓冲队列（输入队列）、装满输出数据的缓冲队列（输出队列）
    • 四种缓冲区：用于收容输入数据的工作缓冲区\(hin\)、用于提取输入数据的工作缓冲区\(sin\)、用于收容输出数据的工作缓冲区\(hout\)、用于提取输出数据的工作缓冲区\(sout\)

  • 按设备的特性分类

    • 独占设备：进程分配到独占设备后，便由其独占，直至该进程释放该设备
    • 共享设备：可同时分配给多个进程，通过分时共享使用
    • 虚拟设备：可将设备同时分配给多个进程

  • 设备分配的数据结构

    • 设备控制表DCT
      • 一个DCT表征一个设备，其表项为设备的各个属性
      • 凡因请求本设备而未得到满足的进程，应将其PCB按某种策略排成一个设备请求队列，设备队列的队首指针指向该请求队列队首PCB
    • 控制器控制表（\(COntroller Control Table,COCT\)）
      • 设备控制器需要请求通道为其服务，因此每个COCT有一个表项存放指向相应CHCT的指针
    • 通道控制表（\(CHannel Control Table,CHCT\)）
      • 一个通道可为多个设备控制器服务，因此CHCT有一个表项指向一个说明所服务的设备控制器的表
      • CHCT与COCT是一对多的关系
    • 系统设备表SDT
      • 整个系统只有一张SDT
      • 记录已连接到系统中的所有物理设备的情况，每个物理设备占一个表目
  • 设备分配原则
    • 设备分配应根据设备特性、用户要求和系统配置情况
    • 既要充分发挥设备的使用效率，又要避免造成进程死锁，还要将用户程序和具体设备隔离开
  • 设备分配方式
    • 静态分配
      • 在用户作业开始执行前，由系统一次性分配该作业所要求的全部设备、控制器
      • 一旦分配，这些设备、控制器就一直为该作业所占用，直到该作业被撤销
    • 动态分配：在进程执行过程中根据执行需要进行、一旦用完，立即释放、可能造成进程死锁
  • 设备分配算法：先请求先分配、优先级高者优先等
  • 设备分配的安全性
    • 安全分配方式
      • 每当进程发出I/O请求后便进入阻塞态，直到其I/O操作完成时才被唤醒
      • 一旦进程己经获得某种设备后便阻塞，不能再请求任何资源，而在它阻塞时也不保持任何资源
    • 不安全分配方式：进程在发出I/O请求后仍继续运行
  • 逻辑设备名到物理设备名的映射
    • 在系统中设置一张逻辑设备表LUT，用于将逻辑设备名映射为物理设备名
    • 其表项包括：逻辑设备名、物理设备名、设备驱动程序入口地址
    • 当进程用逻辑设备名来请求分配设备时，系统为它分配一台相应的物理设备，并在LUT中建立一个表目
    • 当以后进程再利用该逻辑设备名请求I/O操作时，系统通过查找LUT来寻找对应的物理设备和驱动程序
    • 在系统中可采取两种方式设置LUT
      • 在整个系统中只设置一张LUT：所有进程的设备分配情况都记录在同一张LUT中，因此不允许LUT中具有相同的逻辑设备名，主要适用于单用户系统
      • 为每个用户设置一张LUT：每当用户登录时，系统便为该用户建立一个进程，同时也为之建立一张LUT，并将该表放入进程的PCB中
  • \(SPOOLing\)技术（假脱机技术）
    • 在磁盘上开辟出两个存储区域
      • 输入井用于收容I/O设备输入的数据（模拟脱机模拟输入时的磁盘）
      • 输出井用于收容用户程序第二代输出数据（模拟脱机输出时的磁盘）
      • 一个进程的输入（或输出）数据保存为一个文件，所有进程的数据输入（或输出）文件链接成一个输入（或输出）队列
    • 在内存中开辟两个缓冲区
      • 输入缓冲区用于暂存由输入设备送来的数据，之后再传送到输入井
      • 输出缓冲区用于暂存从输出井送来的数据，之后再传送到输出设备
    • 输入/输出进程用于模拟脱机输入/输出时的外围控制机
      • 用户要求的数据从输入设备经过输入缓冲区送到输入井，当CPU需要输入的数据时，直接从输入井读入内存
      • 用户要求输出的数据先从内存送到输出井，待输出设备空闲时，再将输出井中的数据经过输出缓冲区送到输出设备
    • 特点：提高了I/O的速度、将独占设备改造为共享设备、实现了虚拟设备功能（每个进程都认为自己独占了一个设备）

PV专项

基本原理

  • 利用信号量实现同步
    • 初始化一对同步进程的公共信号量S=0
    • 前VP后，即在前操作之后V(S)，在后操作之前P(S)
  • 利用信号量实现互斥
    • 初始化一对互斥进程的信号量S=1（即可用资源数为1）
    • 把临界区置于P(S)和V(S)之间
  • 利用信号量实现前驱关系（多级同步）
    • 为每一对前驱关系的进程设置一个信号量
    • 初始化信号量均为0
    • 前VP后，即在前操作之后V(S)，在后操作之前P(S)

解题思路

  1. 确定函数的个数
    • 梳理题目中有几个进程，一个进程对应一个函数
    • 根据动作是否一致区分是否为统一进程
  2. 确定函数的动作
    • 动作是什么：在函数内部，用中文描述动作（允许用中文的伪代码形式答题）
    • 动作的次数：只做一次（不加while）还是重复进行（while循环）
  3. 确定函数是否在每个动作之前需要进行P操作
    • 如果需要进行P操作，则一定有与之对应的V操作，需要思考这个V操作应该被放在哪进行
    • 消耗资源型的P操作：题目一般会显性给出，例如每次动作需要消耗一个缓冲区空间（P操作减少缓冲区空间，V操作增加缓冲区空间，缓冲区无空间时无法进行动作）
    • 互斥型的P操作：需要注意隐含的互斥关系，例如缓冲区的互斥访问
  4. 确定信号量的个数
    • 所有PV操作定义完成后，再确定信号量的个数
  5. 检查是否发生死锁
    • 连续进行多个P操作的地方是否会发生死锁（只有一个P操作不会发生死锁）
    • 某信号量的PV连续出现（中间没有夹杂别的P操作），则不会发生死锁（不发生请求和保持）
    • 连续多个P导致的死锁可尝试调整P顺序解决

银行家算法

数据结构描述

  • 可利用资源向量Available[m]：每个元素代表一类可用的资源数目
  • 最大需求矩阵Max[n][m]：定义系统中n个进程中每个进程对m类资源的最大需求
  • 分配矩阵Allocation[n][m]：定义系统中n个进程中每个进程已获得m类资源的数目
  • 需求矩阵Need[n][m]：定义系统中n个进程中每个进程最多还需要的m类资源数目
    • Need=Max-Allocation
  • 请求向量Request_i(m)：表示进程i请求m类资源的数目

银行家算法描述

  1. 若\(Request_i[j]\le Need[i,j]\)，则转向步骤2；否则认为出错（请求的资源数超过了之前申明的最大值）
  2. 若\(Request_i[j]\le Available[j]\)，则转向步骤3；否则表示尚无足够资源，进程i需等待
  3. 系统尝试把资源分配给进程\(P_i\)，并修改如下数据结构中的数值
    • \(Available=Available-Request\)
    • \(Allocation[i,j]=Allocation[i,j]+Request_i[j]\)
    • \(Need[i,j]=Need[i,j]-Request_i[j]\)
  4. 系统执行安全性算法，检查此次资源分配后，系统是否处于安全状态
    • 若安全，才正式将资源分配给进程\(P_i\)，完成本次分配
    • 否则，将本次的试探分配作废，恢复原来的资源分配状态，让进程\(P_i\)等待

安全性算法

设置工作向量Work[m]，表示系统中的剩余可用资源数目，在执行安全性算法开始时，Work=Available

  1. 初始时安全序列为空
  2. 从Need矩阵中找出符合下面条件的行，若成功找到，则加入安全序列；否则执行步骤4
    • 该行对应的进程不在安全序列中
    • 该行小于或等于Work向量
  3. 进程\(P_i\)进入安全序列后，可顺利执行，直至完成，并释放被分配的资源，返回步骤2
    • \(Work=Work+Allocation[i]\)
  4. 若此时安全序列中已有所有进程，则系统处于安全状态；否则系统处于不安全状态

磁盘管理

  • 磁盘地址的表示：柱面号·盘面号·扇区号
    • 柱面号在前是为了让寻道尽早开始
  • 磁盘管理
    • 低级格式化（物理格式化）
      • 分为扇区
      • 为每个扇区使用特殊的数据结构（头部、数据区域、尾部），填充磁盘
      • 初始化逻辑块号到无损磁盘扇区的映射
    • 分区（OS完成）
      • 将磁盘分为由一个或多个柱面组成的分区（即C盘、D盘等），每个分区的起始扇区和大小都记录在磁盘主引导记录MBR的分区表中
    • 逻辑格式化（创建文件系统、OS完成）
      • OS将初始的文件系统数据结构存储到磁盘上，包括空闲空间、已分配空间和一个初始为空的目录
      • 每个分区可以是不同的文件系统
      • 为了提高效率，操作系统将多个相邻的扇区组合在一起，形成一簇（在Linux中称为块）
      • 为了更高效地管理磁盘，一簇只能存放一个文件的内容，文件所占用的空间只能是簇的整数倍
  • 逻辑格式化之后灰色部分就有实际数据
  • 主引导记录MBR：包含引导代码和分区表
    • 分区表中有一个表项指向引导分区（包含OS和设备驱动程序）
      • 引导分区的第一个扇区称为引导扇区
  • 坏块：无法正常使用的扇区，硬件故障，OS无法修复
    • 简单磁盘：OS管理，对OS不透明
    • 复杂磁盘：控制器维护，对OS透明
      • 扇区备用：采用备用块来逻辑地替代坏块
  • 一次磁盘读写操作的时间
    • 寻道时间：将磁头移动到指定磁道所需要的时间
    • 旋转延迟时间\(=\frac{1}{2r}\)（r为旋转速度，单位转每秒），即旋转半周的时间
    • 传输时间\(=\frac{b}{rN}\)（b为读/写的字节数，N为一个磁道上的字节数）
  • 磁盘调度算法（减少寻道时间）

  1. 激活CPU：激活的CPU读取ROM中的boot程序，将指令寄存器置为BIOS（基本输入/输出系统）的第一条指令，即开始执行BIOS的指令
  2. 硬件自检：启动BIOS程序后，先进行硬件自检，检查硬件是否出现故障
    • 如有故障，主板会发出不同含义的蜂鸣，启动中止
    • 如无故障，屏幕会显示CPU、内存、硬盘等信息
  3. 加载带有操作系统的硬盘：硬件自检后，BIOS开始读取Boot Sequence（通过CMOS里保存的启动顺序，或者通过与用户交互的方式），把控制权交给启动顺序排在第一位的存储设备

    • 如果发现一个存储设备不是引导盘，就检查下一个存储设备
    • 如无其他启动设备，就会死机
  5. 扫描硬盘分区表，并加载硬盘活动分区
    • MBR包含硬盘分区表，硬盘分区表以特定的标识符区分活动分区和非活动分区

  6. 加载分区引导记录PBR：读取活动分区的第一个扇区（分区引导记录，PBR），其作用是寻找并激活分区根目录下用于引导操作系统的程序（启动管理器）

  8. 加载操作系统

    • 基本概念

      • 协议

        • 规则的集合、对等层之间水平的
        • 组成：语法（格式）、语义（功能、操作）、同步（时序）

      • 接口：服务访问点\(SAP\)

      • 服务

        • 下层向紧邻的上层提供的功能调用、垂直的
        • 服务原语：请求、指示、响应、证实

    • 补充

      • 下一层的协议和所提供的服务的实现细节对上一层透明（看不见）

      • 两台主机通信时，对等层在逻辑上有一条直接信道，表现为不经过下层就把信息传送到对方

      • 并非在一层内完成的全部功能都称为服务，只有那些能够被高一层实体“看得见”的功能才成为服务

物理层

  • 通信基础
    • 数据（模拟数据、数字数据）、信号（模拟信号、数字信号）
    • 码元（信号波形）、\(k\)进制码元、码元宽度
      • 一个\(2^k\)进制的码元可以携带\(k\)比特的数据
    • 信源、信道、信宿
    • 串行传输、并行传输（需要多条链路）
    • 同步传输与异步传输
      • 同步传输
        • 时钟同步：外同步（如单独时钟信号线）、内同步（如曼彻斯特编码）
      • 异步传输：字节之间的时间间隔不固定、需要添加起始位和终止位
    • 交互方式
      • 单工（如广播、仅需一条信道）、半双工（如对讲机、需要两条信道）、全双工（如电话、需要两条信号）
    • 码元传输速率（波特率、调制速率、单位为波特\(Baud\)）、信息传输速率（信息速率、比特率）
      • 若一个码元携带\(n\)比特的信息，则\(M\)波特的码元传输速率对应的信息传输速率为\(M\times n\)比特每秒
  • 奈奎斯特定理：\(\text{理想低通信道下的极限传输速率}=2\cdot W\cdot \log_2{V}\quad (b/s)\)
    • 极限码元传输速率：\(2W\)
    • \(W\)为理想低通信道的带宽（单位赫兹hz）、\(V\)进制码元
    • 采用多元制
  • 香农定理：\(\text{信道的极限数据传输速率}=W\cdot \log_2 {(1+S/N)}\quad (b/s)\)
    • \(W\)为信道的带宽
    • \(S/N\)为信噪比，常以分贝为单位\(=10\cdot \log_{10} {(S/N)}\quad (dB)\)，例如，30dB表示\(S/N=1000\)
    • 提高信噪比
  • \(\min\{\text{奈奎斯特定理},\text{香农定理}\}\)
  • 编码与调制
    • 数字数据编码为数字信号
      • 数字发送器
      • 归零编码\(RZ\)、非归零编码\(NRZ\)、反向非归零编码\(NRZI\)（\(USB 2.0\)）、曼彻斯特编码（以太网使用）、差分曼彻斯特编码、\(4B/5B\)编码（编码效率80%）
      • 对于曼彻斯特编码，“数据传输速率只有调制速率的一半”
    • 数字数据调制为模拟信号
      • 调制解调器
      • 幅移键控\(ASK\)、频移键控\(FSK\)、相移键控\(PSK\)、正交振幅调制\(QAM\)
      • 对于\(QAM\)，若波特率为\(B\)，采用\(m\)个相位，每个相位有\(n\)种振幅，则该技术的数据传输速率为\(R=B\cdot \log_2{(nm)}\quad (b/s)\)
    • 模拟数据编码为数字信号
      • \(PCM\)编码器、音频数字化
      • 步骤：采样、量化、编码
      • \(f_{\text{采样}}\ge 2\times f_{\text{max}}\)
    • 模拟数据调制为模拟信号：放大器
  • 传输介质
    • 导向
      • 双绞线：屏蔽双绞线\(STP\)（有金属丝编织成的屏蔽层）、非屏蔽双绞线\(UTP\)
      • 同轴电缆：基带同轴电缆50\(\Omega\)、宽带同轴电缆75\(\Omega\)
      • 光纤：单模光纤（半导体激光器、远距离、直线传输）、多模光纤（发光二极管、近距离、全反射）
    • 非导向
      • 无线电波
      • 微波、红外线、激光（需要视线通路）
  • 物理层接口特性：机械（引脚数目等）、电气（电压范围等）、功能（意义）、过程（顺序）
  • 物理层设备（不能分割冲突域）
    • 中继器
      • 原理是信号再生、连接两个相同协议的网段、不能连接两个不同速率的局域网
      • 放大器放大模拟信号、中继器再生数字信号
    • 集线器
      • 实质是多端口中继器、转发到除输入端口外的所有端口、逻辑上是总线网
      • 每个端口连接的网路部分是同一个网络的不同网段
      • 只能在半双工下工作（即接受数据和转发数据不能同时进行）
      • 组成的网络是共享式网络、带宽分配
  • 5-4-3规则：在采用粗同轴电缆的10BASE5以太网规范中，5段链路由4个中继器串联，其中只有3段可以挂接计算机
  • 若某个网络设备具有存储转发功能，则可认为其能够连接两个不同的协议

数据链路层

  • 功能

    • 为网络层提供服务

      • 无确认的无连接服务（如以太网）

      • 有确认的无连接服务（如无线通信）

      • 有确认的面向连接服务

      • 有连接就一定要有确认，即不存在无确认的面向连接服务

    • 对于使用点对点信道的DLL

      • 封装成帧：帧定界、帧同步、透明传输（不管所传数据是什么样的比特组合，都应当能在链路上传送）

        • 考虑到差错控制等多种因素，每一种数据链路层协议都规定了帧的数据部分（即IP数据报）的长度上限，即最大传送单元\(MTU\)

      • 差错检测

      • 可靠传输

    • 对于使用广播信道的DLL
      • 介质访问控制MAC：共享式以太网、802.11局域网

  • 封装成帧

    • 字符计数法：计数字段提供的字节数包含自身所占用的一个字节

    • 字符填充的首尾定界符法：用转义字符解决透明传输问题

    • 零比特填充的首尾标志法

      • 用0111 1110标志一帧的开始和结束

      • 信息位中：发送方每遇到5个连续的1就自动在后面插入一个0

    • 违规编码法：如局域网IEEE 802标准采用的曼彻斯特编码方法

  • 差错控制

    • 检错编码
      • 奇偶校验码
        • 使得1的个数为奇数/偶数
      • \(CRC\)循环冗余码

        • 收发双方约定好一个生成多项式G(x)

        • 发送方基于待发送的数据和生成多项式计算出差错检测码（冗余码），将其添加到待传输数据的后面一起传输

        • 接受方通过生成多项式来计算收到的数据是否产生了误码

        • 有很好的检错能力，易于用硬件实现，广泛应用于DLL

        • \(CRC\)循环冗余码也具有纠错功能，但DLL只使用了其检错功能

    • 纠错编码
      • 海明码
        • 海明不等式：信息位n和校验位k应满足
          • \(n+k\le 2^k-1\)
          • 若需检测两位错，则需要再增加一位校验位，即\(k+1\)位
        • 检验d位错：海明码码距为d+1
        • 纠正d位错：海明码码距为2d+1

  • 流量控制：控制发送速率，使得接受方有足够的缓冲空间来接受每一份数据（帧或者分组）

传输层 接收端给发送端一个窗口公告 窗口大小可变

  • 可靠传输

    • \(DLL\)向上层提供的服务类型

      • 不可靠传输服务：仅仅丢弃有误码的帧，其他什么也不做
      • 可靠传输服务：想办法实现发送方发送什么，接收方就收到什么

    • 一般情况下，有限链路的误码率比较低，为了减少开销，并不要求\(DLL\)向上提供可靠传输服务，即使出现了误码，可靠传输的问题由上层处理

    • 无线链路易受干扰，误码率比较高，因此要求\(DLL\)必须向上层提供可靠传输服务

    • 可靠传输并不局限于\(DLL\)，其他各层均可选择实现

  • 可靠传输实现：SW协议、GBN协议、SR协议

  • 接受窗口=1，可保证帧的有序接收

  • DLL的可靠传输通常使用两种机制来完成

    • 确认：一种无数据的控制帧、有时可捎带确认
    • 超时重传：发送方发送某个数据帧后开启一个计时器，在一定时间内未收到对应的确认帧，则重新发送该帧，直至发送成功

  • 自动重传请求ARQ通过接收方请求发送方重传出错的数据帧来恢复出错的帧

  • 连续ARQ协议：当窗口足够大时，帧在信道上可以连续地流动（信道利用率为100%）

  • 窗口大小=发送窗口大小+接收窗口大小

  • 停止等待\(SW\)协议

    • 发送方：发送一份Data（编号0/1交替）、停止发送、缓存、等待对应ACK

      • 若重传时间内收到对应ACK，则发送下一份数据

      • 若超过重传时间仍未收到，则重传

    • 接收方

      • 接受到正确编号、无误码的数据后发送相应ACK
      • 接受到错误编号的数据后丢弃并发送最后接收到的帧对应的ACK
      • 接受到有误码的数据后发送对应NAK

    • 接收端检测到数据分组有误码时，将其丢弃并等待发送方的超时重传。对于误码率较高的链路，为使发送方今早重传，也可给发送方发送NAK

    • 为了使接收方能够判断所收到的数据分组是否是重复的，需要给数据分组编号，由于停等特性，只需一个比特编号

    • 为了让发送方能够判断所收到的ACK分组是否是重复的，需要给ACK分组编号，也只需要一个比特位

    • DLL一般不会出现ACK分组迟到的情况，因此在DLL实现SW协议可以不用给ACK分组编号

    • 超时计时器设置的重传时间略大于从发送方到接收方的平均RTT

    • \(\text{信道利用率}=\frac{\text{数据帧的发送时间}}{\text{数据帧的发送时间}+RTT+\text{确认帧的发送时间}}\)

  • 后退N帧\(GBN\)协议
    • 在流水线传输的基础上利用发送窗口来限制发送方连续发送数据分组的数量，是一种连续ARQ协议
    • 当通信线路质量不好时，其信道利用率并不比SW协议高

  • 介质访问控制MAC：协调多个结点对一个共享传输媒体的占用

  • 静态划分信道：信道划分介质访问控制（多路复用技术）

    • 频分多路复用\(FDM\)

      • 所有用户同时占用不同的频带资源并行通信

    • 时分多路复用\(TDM\)

      • \(TDM\)片划分为多个时隙

      • 所有用户在不同的时间占用同样的频带宽度（类似于并发）

      • 统计时分多路复用\(STDM\)（异步时分多路复用、按需动态分配时隙）

    • 波分多路复用\(WDM\)（光的频分多路复用）

    • 码分多路复用\(CDM\)

      • 采用不同的编码来区分各路原始信号

      • 既共享信道的频率，又共享时间

      • 码分多址\(CDMA\)（理解为点在基内的坐标，向量在各坐标轴上的投影）

        • 任意一个码片向量和该码片向量自身的规格化内积为1
        • 任意一个码片向量和其他码片向量的规格化内积为0
  • 随机访问介质访问控制

    • \(ALOHA\)协议

      • 纯\(ALOHA\)协议

        • 随时发送、若未收到确认则认为发生冲突、等待一段时间后再发送、直至发送成功

        • 吞吐量\(S=G\cdot e^{-2G}\)（G为网络负载，即一段时间内所有站点发送成功和未成功而重传的帧数）、最大0.184

      • 时隙\(ALOHA\)协议

        • 时隙的长度使得每个帧正好在一个时隙内发送完毕

        • 时隙开始时发送

        • 吞吐量\(S=G\cdot e^{-G}\)

    • \(CSMA\)协议：p-<坚持/非坚持>CSMA

      • p表示信道空闲时发送数据的概率
      • <坚持/非坚持>表示信道忙时是否坚持监听
1-坚持CSMA 非坚持CSMA p-坚持CSMA
信道空闲 立即发送数据 立即发送数据 以概率p发送数据，以概率1-p推迟到下一个时隙
信道忙 继续坚持监听 放弃监听，等待一个随机时间后再监听 持续监听，直至信道空闲
备注 若发生冲突，则随机等待一段时间后，再重新开始监听 会增加数据在网络中的平均延迟 折中方案

  • 载波监听多路访问/碰撞检测\(CSMA/CD\)

    • 适用于总线形网络或半双工网络环境
    • 先听后发、边听边发、冲突停发、随机重发
    • 争用期\(2\tau\)（冲突窗口、碰撞窗口）
      • 端到端传播时延的2倍
      • 只有经过争用期还未检测到碰撞时，才能确定本次发送不会发生冲突
    • 最短帧长（争用期内可发送的数据长度）
      • \(\text{最短帧长}=2\times\text{总线单向传播时延}\times \text{数据传输速率}\)
      • 凡长度小于最短帧长的帧都是由于冲突而异常中止的无效帧
      • 若不足最短帧长，则需在MAC子层中的数据字段后面加入填充字段
    • 截断二进制指数退避算法
      1. 确定基本退避时间，一般取\(2\tau\)
      2. 定义参数\(k=\min\{\text{重传次数},10\}\)
      3. 从集合\(\{0,1,2,\cdots ,2^k-1\}\)中随机选取一个数r，重传所需的退避时间即为\(r\times 2\tau\)
      4. 当重传达16次仍不能成功时，说明网络太拥挤，认为此帧永远无法正确发出，抛弃此帧并向高层报告出错

  • CSMA/CD算法归纳

    1. 准备发送，适配器从网络层获得一个分组，封装成帧，放入适配器的缓存

    2. 检测信道

      • 若信道空闲，则开始发送该帧

      • 若信道忙，则持续检测直至信道空闲

    3. 发送过程中，适配器仍持续检测信道

      • 在争用期内一直未检测到碰撞，则发送成功
      • 在争用期内检测到碰撞，则立即停止发送，执行指数退避算法，等待一段随机时间后返回步骤2

    4. 若重传16次仍不能成功，则停止重传并向上报错

  • 无线局域网不能使用CSMA/CD协议

    • 实现碰撞检测的硬件花费很大

    • 存在“隐蔽站”问题

      • 例如，A和C都互相检测不到对方的无线信号

  • 载波监听多路访问/碰撞避免\(CSMA/CA\)
    • 无线局域网
    • 帧间间隔IFS（所有的站点完成发送后，必须再等待一段很短的时间才能发送下一帧，帧间长短取决于该站要发送的帧的类型）

      • SIFS：最短的IFS，用来分割属于一次对话的各帧，使用SIFS的帧类型有ACK帧、CTS帧、分片后的数据帧以及所有回答AP探询的帧等

      • PIFS

      • DIFS：最长的IFS、用于异步帧竞争访问的时延

    • 处理隐蔽站问题：允许发送站对信道进行预约
      • 源站发送数据前先广播请求发送RTS控制帧：（源地址、目的地址、本次通信（含相应的确认帧）所持续的时间）
      • 目的站点（若空闲）广播允许发送CTS控制帧：（本次通信所持续的时间）
        • 给源站明确的发送许可
        • 指示其他站点在预约期内不要发送
  • CSMA/CA协议算法归纳

    1. 若站点最初有数据要发送（而不是发送不成功再进行重传），且检测到信道空闲，在等待时间DIFS后，就发送整个数据帧

    2. 否则，站点执行CSMA/CA退避算法，选取一个随机回退值

      • 一旦检测到信道忙，退避计时器就保持不变（冻结）

      • 只要信道空闲，退避计时器就进行倒计时

    3. 当退避计时器减到0时（这时信道只可能是空闲的），站点就发送整个帧并等到确认

    4. 发送站若收到确认，就知道自己发送的帧被目的站正确接受，此时如果要发送下一个帧，就要从步骤2开始，执行CSMA/CA退避算法，随机选取一段退避时间

    5. 若发送站在规定时间（由重传计时器控制）内没有收到确认帧ACK，就必须重传该帧，再次使用CSMA/CA协议争用信道，知道收到确认，或经过若干次重传失败后放弃发送

  • CSMA/CA协议的基本思想是在发送数据时先广播告知其他结点，让其他结点在某段时间内不要发送数据
  • CSMA/CD协议的基本思想是发送前监听、边发送边监听、一旦出现碰撞马上停止发送
  • 轮询访问介质访问控制
    • 令牌是一个特殊的MAC控制帧
    • 网络空闲时，环路中只有令牌帧在循环传递
    • 令牌传递到有数据要发送的站点时，该站点就修改令牌中的一个标志位，并在令牌中附加自己需要传输的数据，将令牌变成一个数据帧，然后将这个数据帧发送出去
    • 数据帧沿着环路传输，接收到的站点一边转发数据，一边查看帧的目的地址
      • 如果目的地址和自己的地址相同，那么接收站就复制该数据帧以便进一步处理
    • 数据帧沿着环路传输，直到到送孩帧的源站点，源站点收到自己发出去的帧后便不再转发，同时，通过检验返回的帧来查看数据传输过程中是否出错，若有错则重传
    • 源站点传送完数据后，重新产生一个令牌，并传递给下一站点，以交出信道控制权
    • 逻辑拓扑必须是环形、不会发生冲突、非常适合负载很高的广播信道

  • 以太网的\(MAC\)帧（V2标准）

    • （前导码：使接收端与发送端时钟同步）

    • 地址：目的地址+源地址（6B的MAC地址）

    • 类型：指明向上交付的协议实体（802.3帧格式中该字段表示数据域长度）

    • 数据：46$$1500B

    • 填充：使得以太网帧达到64B的最小长度

    • 校验码FCS：CRC循环冗余码、校验范围：除前导码

  • 虚拟局域网\(VLAN\)

    • 把一个较大的局域网分割为一些较小的与地理位置无关的逻辑上的VLAN，而每个VLAN是一个较小的广播域

    • 在以太网帧中插入一个4B的VLAN标签（仅后12位有效，VID的表示范围：\(1\sim 4094\)），用于指明发送方属于的VLAN，插入VLAN标签的帧称为802.1Q帧、插入后FCS必须重新计算

    • 目前使用最广泛的点对点DLL协议

    • 组成

      • 链路控制协议LCP：用于建立、配置以及测试数据链路的连接
      • 网络控制协议NCPs：其中的每一个协议支持不同的网络层协议
      • 一个将IP数据报封装到串行链路的方法
    •

    • 特点

      • 提供差错检测但不提供纠错功能，只保证无差错接受（通过硬件进行CRC校验）
      • 不可靠、不使用序号和确认机制
      • 仅支持点对点、全双工
      • 两端可以运行不同的网络层协议
      • 面向字节
      • 透明传输：异步线路（默认）采用字符填充法；同步线路采用硬件来完成比特填充

  • \(DLL\)设备（隔离碰撞域，但不隔离广播域）

    • 网桥：每个端口连接一个网段
    • 局域网交换机
      • 实质上是多端口网桥、工作在数据链路层、通常工作在全双工方式、可使多对主机同时通信，无碰撞（不使用CSMA/CD协议）
      • 通常每个端口都直接单连一台主机
      • 根据MAC地址对帧进行转发、即插即用、内部的帧转发表通过自学习算法自动建立
      • 一般有多种速率的接口、独占传输媒体带宽
      • 两种交换模式
        • 直通式交换机
          • 只检查帧的目的地址，然后立即转发
          • 速度快、缺乏智能性和安全性、不支持不同速率端口的交换
        • 存储转发式交换机
          • 缓存到高速缓存器、差错检测（误码则丢弃）、转发
          • 可靠性高、支持不同速率端口的转换、延迟较大

  • 交换机的自学习功能

    • 交换表：表项（MAC地址、连通该MAC地址的交换机端口）、过期自动删除、即插即用
    • 对于\(A\to\text{(端口1)交换机(端口2)}\to B\)
      • 若不存在则将(A,1)写入交换表
      • 查找交换表成功则从端口2转发
      • 查找交换表失败则向所有非输入端口广播

  • IPv4分组格式
    • 版本：一般为4
    • 首部长度：单位4B、首部最短20B、最长60B
    • 总长度：首部和数据长度之和、单位1B
    • 用于分片
      • 标识：每个分片的标识字段相同
      • 标志
        • MF（More Fragment）=1表示后面还有分片，=0表示最后一个分片
        • DF（Don't Fragment）=0时才允许分片
      • 片偏移：该片在原分组中的位置、以8B为单位、除最后一个分片外，每个分片的长度一定是8B的整数倍
    • 生存时间TTL：路由器在转发分组之前，先将TTL减1、若TTL等于0，则该分组必须丢弃（同时向源主机发送时间超过ICMP差错报文）
    • 协议：6表示TCP、17表示UDP
    • 首部校验和：只校验分组的首部
    • 源地址字段：发送方IP地址
    • 目的地址字段：接收方IP地址

  • 网际控制报文协议ICMP

    • 让主机或路由器报告差错和异常情况、网络层协议
    • 报文类型
      • ICMP差错报告报文
        • 终点不可达：路由器或主机不能交付数据
        • 源点抑制：路由器或主机由于拥塞而丢弃数据报（最新标准已不再使用）
        • 时间超过：TTL归0、预定时间内分片未全部收到
        • 参数问题：（路由器或目的主机收到的）数据报的首部中有字段值不正确，则丢弃，并发送参数问题报文
        • 改变路由（重定向）：提示主机有更好的路由
      • ICMP询问报文
        • 回送请求和回答报文
        • 时间戳请求和回答报文
        • 地址掩码请求和回答报文
        • 路由器询问和通告报文
    • 不应发送ICMP差错报告报文的情况
      • 对ICMP差错报告报文不发送
      • 对第一个分片的数据报片的所有后续数据报片不发送
      • 对具有组播地址的数据报不发送
      • 对特殊地址（如127.0.0.0、0.0.0.0）的数据报不发送
    • 常见应用
      • 分组网间探测PING：使用ICMP回送请求和回答报文、工作在应用层、直接使用网路层的ICMP（未使用TCP或UDP）
      • Traceroute：使用ICMP时间超过报文、工作在网络层

  • 网络地址转换NAT

    • 将专用网络地址转换为公用地址，对外隐藏内部管理的IP地址
    • 私有IP地址（可重用地址）
      • A类：10.0.0.0$$10.255.255.255
      • B类：172.16.0.0$$172.31.255.255
      • C类：192.168.0.0$$192.168.255.255
    • 因特网中的所有路由器，对目的地址为私有地址的数据报一律不进行转发
    • NAT路由器至少有一个有效的外部全球IP地址
    • NAT转换表中存放着（本地IP地址：端口）到（全球IP地址：端口）的映射
    • 通过这种映射方式，可让多个私有IP地址映射到一个全球IP地址
    • NAT路由器在转发IP数据报时，一定要更换其IP地址（源、目的只更换一个）、工作在传输层
      • 转发到因特网时，修改源IP地址为路由器IP地址
      • 转发到专用网络时，修改目的IP地址为私有IP地址

  • 路由器由于连接多个网络，因此其不仅有多个IP地址，也有多个MAC地址

  • 地址解析协议ARP

    • 完成IP地址到MAC地址的映射、ARP高速缓存、ARP表、工作在网络层
    • 自动进行
      • ARP高速缓存中有，则直接写入MAC帧，发送
      • 没有，则广播目的MAC地址为FFFF-FF-FF-FF-FF的帧来封装ARP请求分组、（真正）接收方单播发送ARP响应分组

  • 路由算法

    • 静态路由算法（非自适应路由算法）：网络管理员手工配置、简单小型的网络
    • 动态路由算法（自适应路由算法）：距离-向量路由算法（如RIP）、链路状态路由算法（如OSPF）

  • 路由信息协议RIP

    • 距离（跳数）：一个路由器到直接连接的网络的距离为1，每经过一个路由器，距离加1
    • 认为好的路由是其通过的路由器的数目少
    • 距离等于16时表示网络不可达、只适用于小型网络
    • 路由器刚开始工作时，仅知道自己与直连网络的距离为1
    • 仅和相邻路由器交换信息
      • 交换的信息是当前路由器所知道的全部信息，即自己的路由表

传输层

  • 传输层功能：端到端的通信（逻辑）、复用（不同应用进程使用同一传输层协议）和分用（正确交付到目的应用进程）、差错检测（首部和数据部分）、提供两种不同的传输协议（面向连接的TCP和无连接的UDP）
  • 网络层也有复用分用功能（通过IP分组首部的协议字段实现分用，6-TCP，17-UDP）
  • 端口：标识主机中的应用进程、只有本地意义
  • 端口号：熟知端口号（\(0\sim 1023\)）、登记端口号（\(1024\sim 49151\)）、客户端使用的端口号（\(49152\sim 65535\)）
  • 套接字\(Socket=(IP\text{地址}:\text{端口号})\)，可以唯一标识网络中的一台主机上的一个应用进程
  • 用户数据报协议UDP
    • 功能：复用、分用、差错检测
    • 无连接、分组首部开销小（8B）、支持单播，组播，广播、不可靠（可靠性由应用层完成）、面向报文（不合并、不拆分、保留报文边界）
    • 首部格式：源端口+目的端口+长度（首部和数据的长度）+校验和（可选、不选则直接置为0）
    • 若目的端口号不正确，则丢弃该报文并由ICMP发送“端口不可达“差错报文
  • 传输控制协议TCP
    • 面向连接、逻辑连接、仅支持单播、可靠传输（无差错、不丢失、不重复、有序）、全双工、面向字节流
    • TCP报文段首部：前20B固定、长度为4B的整数倍
      • 源端口和目的端口
      • 序号seq：该报文段所发送的数据的第一个字节的序号
      • 确认号ack：期望收到对方下一个报文段的第一个数据字节的序号（同时表明前面的已正确收到）
      • 数据偏移（首部长度）：单位4B、TCP首都最大长度为60B
      • 保留
      • 紧急位URG：=1表示应尽快传送
      • 确认位ACK：=1时确认号才有效、在连接建立后，所有传送的报文段都必须把ACK置1
      • 推送位PSH：=1表示应尽快交付给应用进程
      • 复位位RST：=1表示TCP连接出现严重差错，必须立即释放连接，重新建立
      • 同步位SYN：=1表示连接请求或连接接受报文
      • 终止位FIN：=1表示此报文段的发送方的数据已经发送完毕，并要求释放连接
      • 窗口：接收方给发送方的rwnd、单位1B
      • 校验和：检验范围：首部+数据、与UDP基本一致（伪首部中的17改为6）
      • 紧急指针：URG=1时才有意义、本报文段中紧急数据共有多少字节
      • 选项
      • 填充：首部长度为4B的整数倍

  • TCP可靠传输

    • 序号：本报文段所发送的数据的第一个字节的序号
    • 确认：默认采用累积确认
    • 重传
      • 超时：RTP略大于加权平均往返时间\(RTT_s\)
      • 冗余ACK：当发送方收到对同一个报文段的3个冗余ACK（假设\(ack=x\)）时，就认为序号为x及之后的报文段已经丢失，需立即重传（快速重传）

    • 拥塞窗口cwnd：发送方根据对当前网络拥塞程度的估计而确定的窗口值
    • 发送窗口swnd：\(=\min\{rwnd,cwnd\}\)

  • TCP拥塞控制（忽略发送时延）

    • 慢开始（算法）和拥塞避免（算法）
      • 慢开始：初始cwnd=1，每经过一个传输轮次（即RTT），cwnd加倍
        • 当\(2\cdot cwnd>ssthresh\)时，下一个RTT后的\(cwnd=ssthresh\)，即cwnd不能越过ssthresh
        • 每收到一个对新报文段的确认后，将cwnd加1
      • 拥塞避免：\(cwnd>=ssthresh\)时，cwnd线性增加（每经过一个RTT，cwnd加一）
      • 网络出现拥塞时，令\(ssthresh=\frac{1}{2}\cdot cwnd,\quad cwnd=1\)，继续执行慢开始算法
    • 快重传（机制）和快恢复（算法）
      • 快重传：发送方连续收到三个重复的ACK报文时，直接重传接收方尚未收到的报文段，而不必等待重传计时器超时
      • 快恢复：发送方连续收到三个重复的ACK报文时，令\(cwnd=ssthresh=\frac{1}{2}\cdot cwnd\)，之后cwnd线性增加（拥塞避免算法）

  • 在TCP连接建立和网络出现超时时，采用慢开始和拥塞避免算法

  • 当发送方接受到冗余ACK时，采用快重传和快恢复算法

海明码

  1. 确定海明码的位数：\(n+k\le 2^k-1\)
  2. 确定校验位的分布：校验位\(P_i\)在海明位号为\(2^{i-1}\)的位置上，其余为信息位
  3. 分组形成校验关系：被校验数据位的海明位号=校验该数据位的各校验位海明位号之和
  4. 校验位取值：校验位\(P_i\)的值为第i组（由该校验位校验的数据位）所有位求异或
  5. 校验

例如，信息位为\(D_4D_3D_2D_1=1010\)，共4位

  1. \(4+k\le 2^k-1\)得\(k\ge 3\)，因此需要三位校验位\(P_3P_2P_1\)，对应的海明码为\(H_7H_6H_5H_4H_3H_2H_1\)

  2. \(P_1\)的位置为\(2^{1-1}=1\)，即\(H_1\)，同理，\(P_2\)的位置为\(H_2\)，\(P_3\)的位置为\(H_4\)
